Fintech — The Big Picture

Fintech
Technological innovation in the design and delivery of financial services and products. In broader usage, also refers to the companies developing these technologies and the business sector that includes them. Challenging traditional business models of incumbent financial services providers.

Evolution of fintech: Early fintech = data processing + automation of routine tasks → rule-based execution systems → complex machine learning decision-making → advanced AI surpassing human capabilities in specific tasks.

Areas of Fintech Directly Relevant to Investment Analysis

Analysis of Large Datasets
  • Traditional data: security prices, corporate financial statements, economic indicators.
  • Alternative data: social media, sensor networks, satellite imagery.
  • Goal: integrate into portfolio manager's decision-making to generate alpha and reduce losses.
Advanced Analytical Tools
  • AI techniques better suited for complex, non-linear relationships than traditional methods.
  • Sorting company filings, annual reports, earnings calls to identify important data.
  • Uncovering trends and generating insights on human sentiment and behavior.

Big Data and the 4 Vs

Big Data
The vast amount of information being generated by industry, governments, individuals, and electronic devices. Includes both traditional data (stock exchanges, companies, governments) and non-traditional / alternative data (electronic devices, social media, sensor networks, company exhaust). Term in use since the late 1990s.
★ The 4 Vs of Big Data — Core Characteristics
Volume · Velocity · Variety · Veracity
The first three Vs (Volume, Velocity, Variety) describe Big Data itself. The fourth V (Veracity) becomes relevant specifically when Big Data is used for inference or prediction.
V Definition Details Investment Relevance
Volume The amount of data is very large. Millions or even billions of data points. Growing from megabytes/gigabytes to terabytes/petabytes. Traditional analysis methods insufficient; need AI/ML tools. More data → potentially better models if quality is maintained.
Velocity Speed and frequency of data recording/transmission. Real-time or near-real-time data is now the norm. Low-latency (minimal delay) systems required for speed-dependent applications. Critical for high-frequency trading and algorithmic trading strategies. Real-time price feeds, market event data.
Variety Data collected from many sources in many formats. Structured (SQL tables), semistructured (HTML code), and unstructured (video, audio, social media) data. Requires specialized tools (NLP, image recognition) to process unstructured data before analysis.
Veracity Credibility and reliability of different data sources. The 4th V — applies specifically when Big Data is used for inference or prediction. Big Data amplifies the "quality vs. quantity" challenge. Selection bias, missing data, outliers, and data quality must be assessed. Legal/ethical issues with alternative data (e.g., privacy, consent).
🧠 Mnemonic: The 4 Vs

"Very Valuable Ventures Verified" — Volume, Velocity, Variety, Veracity. The first three describe the DATA itself. Veracity is the fourth V that applies when you USE the data for prediction. It's the quality check — "Can we trust it?"

Data Types: Structured, Semistructured, Unstructured

Structured Data
  • Organized in tables (rows and columns).
  • Stored in databases where each field represents the same type of information.
  • Examples: SQL tables, financial databases, trade prices, earnings reports.
  • Traditional analytical tools work directly.
  • Database: SQL, SQLite.
Semistructured Data
  • Has attributes of both structured and unstructured data.
  • Has some organizational tags or markers but doesn't fit a rigid tabular model.
  • Example: HTML code, XML files, JSON data.
  • Requires some processing before analysis but less than fully unstructured.
Unstructured Data
  • Disparate, unorganized — cannot be represented in tabular form.
  • Examples: social media posts, tweets, email, text messages, voice recordings, pictures, videos, blogs, satellite imagery.
  • Requires specialized applications or custom programs before analysis.
  • Database: NoSQL.
  • Most alternative data is unstructured.
⚠️ Exam Trap: Most Big Data / Alternative Data is Unstructured

The majority of alternative data is unstructured and qualitative in nature (texts, photos, videos, audio). This is why traditional analytical methods cannot always be applied directly — specialized tools (AI, ML, NLP) are needed to process unstructured data before any investment analysis can take place. This is also why data curation (cleaning) is so critical.

04

Sources of Big Data: Traditional vs. Alternative

Traditional Data Sources
  • Corporate: annual reports, regulatory filings, sales/earnings figures, conference calls with analysts.
  • Financial markets: trade prices and volumes, exchange data.
  • Economic: government statistics (trade, employment, payroll data).
  • Typically structured, periodic reporting (quarterly, annual).
  • Lagging indicators of performance — reported after the fact.
Alternative Data Sources
  • Non-traditional data from electronic devices, social media, sensor networks, company exhaust.
  • Also called non-traditional data.
  • Mostly unstructured or semi-structured.
  • Can be leading or real-time indicators vs. lagging traditional corporate metrics.
  • Legal/ethical considerations: privacy, consent, regulatory compliance across jurisdictions.

Three Main Sources of Alternative Data

1. Individuals
  • Text, video, photo, audio formats.
  • Website clicks, time spent on webpages.
  • Social media posts, product/service reviews.
  • Web search logs, email, e-commerce data.
  • Typically unstructured. Growing rapidly with online activity.
2. Business Processes
  • Information flows from corporations and public entities.
  • Credit card / transaction data, direct sales data.
  • Corporate exhaust: supply chain info, banking records, retail POS scanner data.
  • Typically structured.
  • Can be leading or real-time performance indicators vs. lagging traditional metrics.
3. Sensors
  • Smart phones, cameras, RFID chips, satellites.
  • Connected via wireless networks.
  • Satellite imagery: retail parking lots, shipping, agriculture, oil rigs.
  • Can be unstructured. Volume is orders of magnitude larger than individual/business data.
  • Internet of Things (IoT): smart buildings, vehicles, appliances sharing data.
💡 Alternative Data Investment Applications

From retail sales data to social media sentiment to satellite imagery — alternative data provides insights into consumer behavior, firm performance, and market trends that traditional data cannot. Examples: satellite images of parking lots count shoppers before earnings announcements; social media sentiment predicts IPO performance; credit card transaction data reveals real-time spending trends ahead of quarterly reports.

⚠️ Legal & Ethical Issues with Alternative Data

Web scraping can capture personally protected information. Data may have been published without the explicit knowledge or consent of individuals. Best practices are still developing across jurisdictions; regulatory approaches can conflict. Investment professionals must assess legal and ethical dimensions before using alternative data in investment decisions.

Artificial Intelligence & Machine Learning

Artificial Intelligence (AI)
Computer systems capable of performing tasks that traditionally required human intelligence, at levels comparable to or superior to those of human beings. Encompasses decision-making ability, cognitive tasks, pattern recognition, and reasoning. Historical example: expert systems using "if-then" rules. Now includes neural networks, deep learning, NLP.
Machine Learning (ML)
A subset of AI. Computer-based techniques that extract knowledge from large amounts of data without making assumptions about the data's underlying probability distribution. Goal: automate decision-making by generalizing ("learning") from known examples to determine underlying structure. Simply put: "Find the pattern, apply the pattern."

Why ML requires Big Data: ML algorithms require massive amounts of data for "training." Historically, insufficient data limited ML applications. The growth of Big Data has provided ML (especially neural networks) with sufficient data to improve modeling and predictive accuracy. Without enough data, the model cannot train and validate properly.

How ML Works — The Training Process

IN
Algorithm is given inputs (a set of variables or datasets) and possibly outputs (target/labeled data, in supervised learning). The "inputs" are the features/predictors; the "outputs" are what the model is trying to predict.
TR
Training dataset: algorithm identifies relationships between inputs and outputs based on historical patterns. The model parameters are fitted/optimized on this data. This is where "learning" occurs.
VL
Validation dataset: relationships learned in training are validated and the model is tuned/refined. Used to prevent overfitting and select optimal model complexity. Parameters are not fitted again — only model configuration is adjusted.
TS
Test dataset: final assessment of the model's ability to predict well on completely new, unseen data. The gold standard — provides an unbiased estimate of real-world performance. Data not used in training or validation.
→
Once trained, validated, and tested — the ML model can be deployed to predict outcomes based on other new datasets.
💡 ML Still Requires Human Judgment

Despite automation, ML is not fully autonomous. Humans must: (1) understand the underlying data, (2) select appropriate ML techniques, (3) ensure data is clean and free of biases, (4) have sufficient data for training and validation, (5) interpret results and catch "black box" outputs that may not be explainable. ML is a tool — human expertise guides its application.

06

Types of Machine Learning

Supervised Learning
  • Computers learn to model relationships from labeled training data — both inputs AND outputs are identified for the algorithm.
  • After training, used to predict outcomes for new data.
  • Investment examples:
    • Predicting next-day stock return direction (up/down/flat).
    • Identifying the best signal/variable for forecasting returns.
    • Credit scoring, fraud detection.
Unsupervised Learning
  • Computers are given only data — no labeled outputs. Algorithm seeks to describe the data and its underlying structure.
  • Goal: discover hidden patterns, groupings, or structure in data.
  • Investment examples:
    • Grouping companies into peer groups based on financial characteristics (rather than standard sector classifications).
    • Clustering securities by risk/return profile.
    • Anomaly detection.
Deep Learning
  • Uses neural networks with many hidden layers for multistage, non-linear data processing.
  • Can use supervised OR unsupervised approaches.
  • Layered approach: simple concepts inform analysis of more complex concepts.
  • Investment examples:
    • Image recognition from satellite data.
    • Speech recognition (earnings calls).
    • Pattern recognition in large financial time series.
Summary: Three ML Types
Supervised: Labeled inputs + outputs → predict new outputs from new inputs
Unsupervised: Only inputs (no labels) → discover structure/patterns in data
Deep Learning: Neural networks + hidden layers → multi-stage non-linear analysis
                 (can use supervised or unsupervised approach)
Neural networks (underlying deep learning) have existed since 1958. Improvements in algorithms provide more accurate models requiring less computing power than earlier versions. Better at image, pattern, and speech recognition. Neural network basics: programming based on how our brain learns and processes information.
🧠 Mnemonic: Supervised vs. Unsupervised

Supervised = Teacher is present — the algorithm is given both the questions (inputs) AND the answers (labeled outputs), like a student learning from an answer key. Unsupervised = No teacher — only data is given, the algorithm must figure out the structure on its own, like sorting a pile of objects into groups without being told the categories. Deep learning = many layers of neurons, like peeling an onion one layer at a time.

Overfitting, Underfitting & the Black Box Problem

⚠️ Overfitting

The ML model learns the training data too precisely — it has been "overtrained." It treats noise in the data as true parameters. Result: the model discovers "false" relationships or "unsubstantiated" patterns → prediction errors on new data. The model is too complex. Cannot generalize to new datasets. This is the most tested Big Data pitfall on the exam.

⚠️ Underfitting

The ML model treats true parameters as if they are noise — it fails to recognize real relationships within the training data. Result: misses patterns that underlie the data. The model is too simplistic. Also produces poor predictions but for the opposite reason — not enough model complexity to capture the true relationship.

IssueWhat HappensModel ComplexityTraining PerformanceNew Data Performance
Overfitting Model learns noise as signal; "false" patterns discovered Too complex Excellent (fits training data perfectly) Poor (fails to generalize)
Underfitting Model misses true patterns; true signal treated as noise Too simple Poor (can't capture even the training data well) Poor (missed patterns)
Good Fit Model captures true relationships without memorizing noise Appropriate Good Good (generalizes well)
"Black Box" Problem
Because ML techniques are not explicitly programmed with rules, they can appear opaque — they arrive at outcomes that might not be entirely understood or explainable by humans. The model works, but why it works is not always transparent. This is a significant concern in regulated investment management contexts where explainability and auditability of decisions is required.
💡 Why the Train / Validation / Test Split Prevents Overfitting

The three-way data split is the key defense against overfitting. Training data fits the model. Validation data checks whether the fitted model generalizes — if performance drops sharply on validation vs. training, the model is overfitting. Test data provides final, unbiased performance assessment on completely unseen data. Never use test data during model development — its purity is what makes the final evaluation meaningful.

Data Science & Data Processing Methods

Data Science
An interdisciplinary field that harnesses advances in computer science (including ML), statistics, and other disciplines for the purpose of extracting information from Big Data (or data in general). Data scientists/analysts extract information and insights for business and investment purposes.

Five Data Processing Methods (CSSST)

1
Capture — How data are collected and transformed into a usable format. Low-latency systems (minimal delay, high-volume data communication) are essential for automated trading. High-latency systems do not require real-time data.
2
Curation — Ensuring data quality and accuracy through a data cleaning exercise. Reviewing all data to detect errors, bad/inaccurate data, and making adjustments for missing data. Critical before any analysis — "garbage in, garbage out."
3
Storage — How data will be recorded, archived, and accessed. Underlying database design. Key consideration: structured vs. unstructured data, and whether low-latency solutions are needed for real-time access.
4
Search — How to query data. Big Data has created the need for advanced applications capable of examining and reviewing large quantities of data to locate requested content. Simple SQL queries are insufficient for massive unstructured datasets.
5
Transfer — How data moves from the source/storage location to the analytical tool. Could be through a direct data feed (e.g., a stock exchange's price feed) or batch file transfer.
🧠 Mnemonic: Five Data Processing Steps

"Can Clever Students Store Text?" — Capture, Curation, Storage, Search, Transfer. The order roughly follows the data lifecycle: collect it → clean it → store it → find what you need → move it to your analytical tool.

Data Visualization

An important tool for understanding Big Data. Visualization = how data is formatted, displayed, and summarized in graphical form.

  • Traditional structured data: tables, charts, trend lines.
  • Unstructured data: interactive 3D graphics, multidimensional charts with color/shapes/sizes, heat maps, tree diagrams, network graphs.
  • Tag cloud: words sized by frequency — words appearing more often shown in larger font. Applicable to textual data. Key words in module: "data," "ML," "learning," "AI," "techniques."
  • Mind map: variation of tag cloud — shows how different concepts are related to each other (rather than just word frequency).

Text Analytics & Natural Language Processing (NLP)

Text Analytics
Use of computer programs to analyze and derive meaning from large, unstructured text- or voice-based datasets (company filings, reports, earnings calls, social media, email, surveys). Includes automated information retrieval, lexical analysis (word frequency), pattern recognition based on key words/phrases, and predictive analysis for consumer sentiment.
Natural Language Processing (NLP)
A field at the intersection of computer science, AI, and linguistics. Focuses on developing programs to analyze and interpret human language. The most important application within text analytics. Can scale analysis across thousands of companies globally — work previously humanly impossible.

NLP Tasks and Investment Applications

NLP Automated Tasks
  • Translation
  • Speech recognition
  • Text mining
  • Sentiment analysis
  • Topic analysis
  • Compliance monitoring: detecting fraud, reviewing employee communications for policy adherence, protecting confidential information.
Investment Management Applications
  • Monitoring analyst commentary → detect sentiment shifts before recommendation changes are formalized.
  • Analyzing central bank (Fed, ECB) transcripts → detect subtle shifts in tone on interest rates, inflation, output.
  • Predicting IPO performance from Twitter sentiment.
  • Analyzing annual reports and earnings calls at scale (millions of pages → beyond human capacity).
  • Identifying short-term trend indicators from social media / news sentiment.
💡 Why NLP Matters for CFA: Analyst Sentiment Example

Analysts rarely change buy/hold/sell recommendations explicitly — instead they embed nuanced commentary in written reports. NLP can assign sentiment ratings (very negative → very positive) to these reports and detect shifts in tone before a formal recommendation change. This gives investors a potential informational edge. The ability to scale this across thousands of companies worldwide is only possible through NLP + ML — not humanly possible otherwise.

Programming Languages & Databases

Key Programming Languages

LanguageKey FeaturesPrimary Use in Finance/Fintech
Python Open-source, free, accessible to non-programmers. Low learning curve. Basis for many fintech applications. Advanced analytics, ML models, data processing. Most widely used in finance data science.
R Open-source, free. Traditional statistical analysis language. Rich mathematical packages. Statistical analysis, ML, optimization, econometrics, financial analysis. Used heavily in academic research.
Java Runs on different computers, servers, and operating systems (platform-independent). Underlying language for many internet applications. Enterprise-level financial systems.
C and C++ Specialized. Optimize source code for superior calculation speed and processing performance. Algorithmic trading and high-frequency trading (HFT) — speed is critical.
Excel VBA Bridges programming and manual data processing. Run macros to automate tasks. Automating data tables, formulas, web data queries, customized reports. Gateway to automation for non-programmers.

Key Database Types

SQL

Database query language for structured data. Data stored in tables with rows and columns. Run on a server; accessed via SQL queries. Standard for traditional financial databases.

SQLite

Database for structured data. Embedded into the program — does NOT require a server. Most common database for mobile apps requiring data access. Lightweight version of SQL.

NoSQL

Database for unstructured data. Data cannot be summarized in traditional rows/columns. Required when data is too flexible, diverse, or large for relational tables. Used for Big Data / alternative data storage.

🧠 Quick Reference: Data Type → Database Match

Structured data → SQL or SQLite (tables, rows, columns). Unstructured data → NoSQL (flexible, no fixed schema). Remember: "No SQL" = "No table structure required." C/C++ = speed for HFT. Python = most versatile for fintech analytics.
