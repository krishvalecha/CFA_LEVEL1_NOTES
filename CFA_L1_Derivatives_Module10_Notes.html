01

Why We Need the Binomial Model — The Gap Forward Pricing Can't Fill

Forward commitments can be priced purely using no-arbitrage and replication — no assumptions about future prices are needed because the payoff is symmetric and certain.

Options and contingent claims are different. Their payoff is asymmetric — they only settle when in the money. To value them, you must make assumptions about how the underlying price can evolve over time. The binomial model provides the simplest possible framework: the underlying can only go up or go down over a single period. This is enough to derive a no-arbitrage option price.

Forward Pricing — No Model Needed
  • Symmetric, certain payoff: S_T − F₀(T)
  • Priced by replication: borrow + buy underlying
  • No assumption about future prices required
  • F₀(T) = S₀(1+r)^T
Option Pricing — Model Required
  • Asymmetric payoff: Max(0, S_T − X)
  • Payoff depends on whether S_T > X or not
  • Must assume a price distribution for the underlying
  • Binomial model: simplest valid assumption
💡 Key Insight — The Binomial Model's Power

Despite its simplicity (only two possible outcomes!), the one-period binomial model correctly prices options using no-arbitrage. Extended to many periods, it converges to the Black-Scholes-Merton model. The exam focuses on the one-period version — understanding the logic deeply here unlocks all option pricing theory.

02

Binomial Model Setup — The Two-State World

The binomial model assumes that over a single period (from t=0 to t=T), the underlying asset price will do exactly one of two things: move up to S₁ᵘ or move down to S₁ᵈ. Nothing else is possible. This two-state assumption is what makes the model tractable.

Binomial Model — Core Variables
S₀ = current underlying price (known at t=0)
S₁ᵘ = underlying price if UP move occurs = Rᵘ × S₀
S₁ᵈ = underlying price if DOWN move occurs = Rᵈ × S₀

Rᵘ = up gross return = S₁ᵘ / S₀ > 1 (price rises)
Rᵈ = down gross return = S₁ᵈ / S₀ < 1 (price falls)

q = real-world (actual) probability of up move
1−q = real-world probability of down move

KEY: The spread (S₁ᵘ − S₁ᵈ) represents VOLATILITY.
Larger spread = higher volatility = higher option value.
What determines Rᵘ and Rᵈ? They should be calibrated to match the observed or expected volatility of the underlying asset. Higher volatility → wider spread between S₁ᵘ and S₁ᵈ → Rᵘ further above 1, Rᵈ further below 1.

Requirement: S₁ᵈ < X < S₁ᵘ for a standard call/put example (exercise price between the two possible outcomes). If both S₁ᵘ and S₁ᵈ are above X, the call is always ITM; if both are below, it's always OTM — trivial cases.

Binomial Tree — Visual Representation

One-Period Binomial Tree (European Call, X = €100 example)
t = 0
S₀ = €80
c₀ = ?
→
UP: S₁ᵘ = €110 (Rᵘ = 1.375)
c₁ᵘ = Max(0, 110−100) = €10 ✓ ITM
DOWN: S₁ᵈ = €60 (Rᵈ = 0.75)
c₁ᵈ = Max(0, 60−100) = €0 ✗ OTM
Key assumption: S₁ᵈ = €60 < X = €100 < S₁ᵘ = €110
c₁ᵘ and c₁ᵈ are KNOWN at t=0 (function of S₁ᵘ, S₁ᵈ, and X)
Only c₀ is UNKNOWN — this is what we solve for.
💡 The Critical Observation — Option Values at T are Already Known

At option expiration (t=T), the option value is simply Max(0, S_T − X) for a call or Max(0, X − S_T) for a put. Since we know S₁ᵘ and S₁ᵈ at t=0, we can calculate c₁ᵘ and c₁ᵈ at t=0 — they are NOT uncertain. The only unknown is c₀ (the option price today). The binomial model gives us a way to solve for this unknown.

03

Constructing the Risk-Free Hedge Portfolio

The key idea: combine the option with the underlying asset in exactly the right proportion so that the combined portfolio has the same value whether the price goes up or down. This creates a risk-free portfolio — and a risk-free portfolio must earn the risk-free rate. This no-arbitrage condition lets us solve for the option price.

The Hedge Portfolio — Structure
At t=0: SELL one call option (receive c₀) + BUY h units of underlying (pay h×S₀)

Portfolio value at t=0: V₀ = h×S₀ − c₀

Portfolio value at t=1:
If UP: V₁ᵘ = h×S₁ᵘ − c₁ᵘ = h×Rᵘ×S₀ − Max(0, S₁ᵘ−X)
If DOWN: V₁ᵈ = h×S₁ᵈ − c₁ᵈ = h×Rᵈ×S₀ − Max(0, S₁ᵈ−X)

GOAL: Choose h such that V₁ᵘ = V₁ᵈ (same value in BOTH scenarios)
→ The portfolio becomes risk-free (value doesn't depend on which scenario occurs)
Why sell the call and buy the underlying? The call increases in value when the underlying rises (creating a gain for the call buyer / loss for the seller). By buying the underlying, we create an offsetting gain when the price rises. Choosing h correctly makes these exactly cancel — creating a riskless combination.

Sign convention: Sold call = negative (liability). Purchased underlying = positive (asset). V₀ = net investment by the seller of the call to hedge their position.
From the European call example: S₀=€80, X=€100, S₁ᵘ=€110, S₁ᵈ=€60, c₁ᵘ=€10, c₁ᵈ=€0
V₁ᵘ = h × 110 − 10
V₁ᵈ = h × 60 − 0 = 60h

Set V₁ᵘ = V₁ᵈ:
110h − 10 = 60h
50h = 10
h = 0.20

Verify: V₁ᵘ = 110 × 0.20 − 10 = 22 − 10 = €12
V₁ᵈ = 60 × 0.20 − 0 = €12 ✓ Both equal €12 (risk-free!)
⚠️ The Risk-Free Portfolio Must Earn the Risk-Free Rate — No More, No Less

Once the portfolio is risk-free (V₁ᵘ = V₁ᵈ = V₁), it must earn exactly the risk-free rate r. If it earned more: buy the portfolio and sell a risk-free bond → riskless profit. If it earned less: sell the portfolio and buy a risk-free bond → riskless profit. So V₁ = V₀ × (1+r), which means V₀ = V₁ / (1+r). This no-arbitrage condition is what pins down c₀.

04

The Hedge Ratio — Formula & Intuition (Equation 6)

Equation 6 — Hedge Ratio (h*)
h* = (c₁ᵘ − c₁ᵈ) / (S₁ᵘ − S₁ᵈ)

WHERE:
c₁ᵘ = option value if underlying goes UP [= Max(0, S₁ᵘ−X) for call]
c₁ᵈ = option value if underlying goes DOWN [= Max(0, S₁ᵈ−X) for call]
S₁ᵘ = underlying price if UP
S₁ᵈ = underlying price if DOWN

All values on the RIGHT-HAND SIDE are known at t=0.
h* is the number of underlying units to buy per option sold.
Intuition: h* = (range of option values) / (range of underlying prices). It measures how much the option moves per unit move in the underlying. This is essentially the option's delta (Δ) — the concept introduced informally in Module 8.

For a call: h* is always between 0 and 1. Deep ITM: h* → 1. ATM: h* ≈ 0.5. Deep OTM: h* → 0.
For a put: h* = (p₁ᵘ − p₁ᵈ) / (S₁ᵘ − S₁ᵈ). Since the put gains when S falls, p₁ᵘ < p₁ᵈ → h* is negative (you BUY the put AND the underlying, or equivalently, the put replication requires a short underlying position).

In practice: h* is expressed as a fraction. To hedge using whole contracts, scale up proportionally (e.g., h*=0.25 means sell 4 calls per underlying unit purchased, or buy 100 underlying units for every 400 call options sold).
💡 Hedge Ratio = Delta — The Link to Module 8

In Module 8, we said option replication requires purchasing a "proportion" of the underlying based on the likelihood of exercise. That proportion IS the hedge ratio h* from the binomial model. For a call, h* = (c₁ᵘ − c₁ᵈ) / (S₁ᵘ − S₁ᵈ), which equals the option's delta Δ. The higher the probability of exercise (i.e., deeper ITM), the higher h* — meaning you need to hold more of the underlying to hedge. This is why ATM options (h* ≈ 0.5) require 50% of the underlying, while deep ITM options (h* → 1) require nearly the full position.

05

Solving for the Option Price — The No-Arbitrage Method

Once the hedge ratio h* is known and the risk-free portfolio value V₁ is computed, the option price c₀ follows directly from the no-arbitrage condition that the portfolio must earn the risk-free rate.

Step-by-Step: Solving for c₀ via No-Arbitrage
STEP 1: Compute option payoffs at t=1
c₁ᵘ = Max(0, S₁ᵘ − X) [call up payoff]
c₁ᵈ = Max(0, S₁ᵈ − X) [call down payoff]

STEP 2: Compute hedge ratio h*
h* = (c₁ᵘ − c₁ᵈ) / (S₁ᵘ − S₁ᵈ)

STEP 3: Compute risk-free portfolio value at t=1
V₁ = h* × S₁ᵘ − c₁ᵘ [or use down scenario: h* × S₁ᵈ − c₁ᵈ — same result]

STEP 4: Discount to t=0
V₀ = V₁ / (1+r) [since the portfolio is risk-free, discount at r]

STEP 5: Solve for c₀
V₀ = h* × S₀ − c₀
c₀ = h* × S₀ − V₀

FINAL FORMULA:
c₀ = h* × S₀ − V₁ × (1+r)^−1
Logic of Step 5: At t=0, the hedged portfolio costs V₀ = (h* × S₀) − c₀. We just computed what V₀ must be (it's the PV of V₁). Rearranging: c₀ = h* × S₀ − V₀. So the option price equals the cost of the hedge position (buying h* units of underlying at S₀) minus the fair value of the combined portfolio.

Confirmation: V₀ × (1+r) = V₁ should hold. If it does, the hedge portfolio earns exactly r → no-arbitrage is satisfied.

Full Worked Example — European Call (from text)

S₀=€80 | X=€100 | S₁ᵘ=€110 | S₁ᵈ=€60 | r=5% per year | T=1 year
STEP 1: Option payoffs at T
c₁ᵘ = Max(0, 110−100) = €10
c₁ᵈ = Max(0, 60−100) = €0

STEP 2: Hedge ratio
h* = (10 − 0) / (110 − 60) = 10/50 = 0.20
→ For each call sold, buy 0.20 units of underlying (or sell 5 calls per 1 unit of underlying)

STEP 3: Risk-free portfolio value at T
V₁ = h* × S₁ᵘ − c₁ᵘ = 0.20 × 110 − 10 = 22 − 10 = €12
Check: 0.20 × 60 − 0 = €12 ✓

STEP 4: Discount to t=0
V₀ = 12 / 1.05 = €11.43

STEP 5: Solve for c₀
c₀ = h* × S₀ − V₀ = 0.20 × 80 − 11.43 = 16.00 − 11.43 = €4.57

CONFIRMATION: V₀ × (1+r) = 11.43 × 1.05 = €12.00 = V₁ ✓
06

Risk-Neutral Pricing — The Elegant Shortcut (Equation 7)

The hedge portfolio method works but involves multiple steps. There is a mathematically equivalent — and often faster — approach: risk-neutral pricing. Instead of building a hedge portfolio, we use specially constructed "risk-neutral probabilities" to compute the expected option payoff, then discount at the risk-free rate.

The result is identical to the hedge portfolio method. The risk-neutral approach gives us a clean formula for option pricing that can be applied to any European option — call or put — using the same pair of probabilities.

Equation 7 — Risk-Neutral Probability of an Up Move (π)
π = (1 + r − Rᵈ) / (Rᵘ − Rᵈ)

WHERE:
r = risk-free rate (per period)
Rᵘ = up gross return = S₁ᵘ / S₀
Rᵈ = down gross return = S₁ᵈ / S₀

(1 − π) = risk-neutral probability of a DOWN move

Key constraint for no-arbitrage: Rᵈ < (1+r) < Rᵘ
(The risk-free return must be between the up and down gross returns)
Where does this come from? π is derived by setting the risk-neutral expected return of the underlying equal to the risk-free rate: π×Rᵘ + (1−π)×Rᵈ = (1+r). Solving for π gives the formula above. This is NOT the real-world probability q of an up move. It's a synthetic probability that makes every asset earn the risk-free rate — used purely as a pricing tool.

No-arbitrage constraint: If Rᵈ ≥ (1+r), you'd borrow at r and invest in the down-move asset for a riskless profit. If Rᵘ ≤ (1+r), the underlying always underperforms the risk-free rate — also impossible to sustain. So Rᵈ < (1+r) < Rᵘ must hold.
Risk-Neutral Option Pricing Formula — Any European Option
c₀ = [π × c₁ᵘ + (1−π) × c₁ᵈ] / (1+r)^T

p₀ = [π × p₁ᵘ + (1−π) × p₁ᵈ] / (1+r)^T

More generally, for ANY European derivative (D₀):
D₀ = [π × D₁ᵘ + (1−π) × D₁ᵈ] / (1+r)^T

WHERE:
D₁ᵘ, D₁ᵈ = derivative payoffs at T in up and down scenarios
π, (1−π) = risk-neutral probabilities (same for ALL options on same underlying)
r = risk-free rate
T = time to expiration
This formula is universal. The risk-neutral probabilities π and (1−π) are properties of the UNDERLYING ASSET (determined by Rᵘ, Rᵈ, and r). They are NOT specific to any particular option. So once you compute π for a given underlying and volatility assumption, you can use the same π to price any call or put on that underlying — regardless of exercise price or maturity structure.

Risk-Neutral Pricing — Worked Calculation (Same Example)

S₀=€80 | X=€100 | Rᵘ=1.375 | Rᵈ=0.75 | r=5%
STEP 1: Compute π
π = (1 + r − Rᵈ) / (Rᵘ − Rᵈ)
= (1 + 0.05 − 0.75) / (1.375 − 0.75)
= 0.30 / 0.625
= 0.48
(1−π) = 0.52

STEP 2: Compute risk-neutral expected payoff
c₀ numerator = π × c₁ᵘ + (1−π) × c₁ᵈ
= 0.48 × €10 + 0.52 × €0
= €4.80

STEP 3: Discount at risk-free rate
c₀ = 4.80 / 1.05 = €4.57 ✓ matches hedge portfolio method
★ Two Methods, Same Answer — Always

The hedge portfolio method and the risk-neutral pricing method are mathematically equivalent. They always produce the same option price. The risk-neutral method is generally faster (fewer steps, one formula) and is more powerful in multi-period models and continuous-time frameworks (like Black-Scholes-Merton). For the exam, know both methods and be able to confirm the result using the other.

07

Pricing a Put Option — Same Framework, Same π

The binomial model works identically for put options. The only differences are: (1) the payoffs at T are Max(0, X − S_T) instead of Max(0, S_T − X), and (2) the hedge ratio h* will be negative (because you must hold short underlying to hedge a long put). The risk-neutral pricing formula is exactly the same — just substitute put payoffs.

Pricing a Put Using Risk-Neutral Method
Put payoffs at T:
p₁ᵘ = Max(0, X − S₁ᵘ) [up scenario: usually OTM for a put]
p₁ᵈ = Max(0, X − S₁ᵈ) [down scenario: usually ITM for a put]

Risk-neutral put price:
p₀ = [π × p₁ᵘ + (1−π) × p₁ᵈ] / (1+r)^T

Use SAME π as computed for the call on the same underlying.
(π is a property of the underlying, not of the specific option.)
💡 Shortcut — Use Put–Call Parity After Computing Call

If you've already computed c₀ via the binomial model, you don't need to redo the full analysis for p₀. Just use put–call parity:
p₀ = c₀ − S₀ + X(1+r)^−T
This is faster and guaranteed to give the same answer as the binomial model (both satisfy no-arbitrage). The exam frequently tests both approaches and asks you to confirm consistency between them.

08

Volatility, Real Probabilities & Key Conceptual Insights

Insight 1: Real-World Probabilities (q) Don't Matter

✅ The Option Price is Independent of q (Real-World Probability)

The actual (real-world) probability of an up or down move — denoted q — does NOT affect the no-arbitrage option price. The option price depends only on:

  • The up and down gross returns (Rᵘ and Rᵈ) — i.e., the magnitude of price moves (= volatility)
  • The risk-free rate (r)
  • The exercise price (X) and current underlying price (S₀)

Even if you're highly optimistic about the stock (q = 0.9) or pessimistic (q = 0.1), the no-arbitrage call price is the same. Why? Because anyone with a different opinion can construct a hedge using h* units of the underlying — and the hedged portfolio is risk-free regardless of the actual probability of each outcome.

⚠️ Common Exam Trap — "More Optimistic Outlook → Higher Option Price"

WRONG. A more optimistic probability of an up move (higher q) does NOT change the no-arbitrage option price. The binomial model's power is precisely that it doesn't require knowledge of real-world probabilities. The option price is uniquely determined by the no-arbitrage/hedge portfolio condition, which is independent of q. Only the SIZE of up/down moves (Rᵘ, Rᵈ) — i.e., volatility — matters, not the probability of each move.

Insight 2: Higher Volatility → Higher Option Value

Volatility in the binomial model = the spread between S₁ᵘ and S₁ᵈ (or equivalently, how far Rᵘ is above 1 and how far Rᵈ is below 1). A wider spread always increases option value — for both calls and puts:

  • Call: Wider spread → S₁ᵘ is further above X → c₁ᵘ is larger → call is worth more
  • Put: Wider spread → S₁ᵈ is further below X → p₁ᵈ is larger → put is worth more

This is the binomial model's way of capturing what we showed qualitatively in Module 8: volatility (σ) always increases both call and put values.

Volatility AssumptionPrice Rangeh* (Call)c₀ (Call)p₀ (Put)
Low (±20%)$40 to $600.25$2.98$5.36
High (±40%)$30 to $700.375$8.04$10.42
From Hightest Capital Example 1: S₀=$50, X=$55, r=5%. Higher volatility → both call and put values increase significantly.

Insight 3: No-Arbitrage Constraint — Rᵈ < (1+r) < Rᵘ

For risk-neutral probabilities to be valid (between 0 and 1), the risk-free rate must fall strictly between the down and up gross returns. This is required for no-arbitrage:

  • If Rᵈ ≥ (1+r): Even the down move earns at least the risk-free rate → borrow at r, invest in asset → riskless profit
  • If Rᵘ ≤ (1+r): Even the up move earns at most the risk-free rate → sell the asset, invest proceeds at r → riskless profit

Insight 4: Same π for All Options on the Same Underlying

💡 One Set of Risk-Neutral Probabilities for All Options

The risk-neutral probabilities π and (1−π) are determined by Rᵘ, Rᵈ, and r — which are properties of the underlying asset and the binomial model structure. They are the same regardless of the exercise price X or whether you're pricing a call or a put. Once you compute π for a given underlying/volatility setup, use it to price ANY option on that underlying. This makes risk-neutral pricing extremely efficient in multi-option problems.

Full Worked Examples — Hightest Capital (Examples 1 & 2)

Example 1 — Call Pricing via Hedge Portfolio (±20% volatility)

S₀=$50 | X=$55 | r=5% | T=1yr | ±20% price move
STEP 1: Underlying prices at T
S₁ᵘ = 50 × 1.20 = $60 | S₁ᵈ = 50 × 0.80 = $40
Rᵘ = 1.20, Rᵈ = 0.80

STEP 2: Call payoffs at T
c₁ᵘ = Max(0, 60−55) = $5 | c₁ᵈ = Max(0, 40−55) = $0

STEP 3: Hedge ratio
h* = (5 − 0) / (60 − 40) = 5/20 = 0.25
(Sell 1 call, buy 0.25 units of underlying; or sell 4 calls per 1 underlying unit)

STEP 4: Risk-free portfolio value at T
V₁ = h* × S₁ᵘ − c₁ᵘ = 0.25 × 60 − 5 = 15 − 5 = $10
Check: 0.25 × 40 − 0 = $10 ✓

STEP 5: Discount to t=0
V₀ = 10 / 1.05 = $9.524

STEP 6: Solve for c₀
c₀ = h* × S₀ − V₀ = 0.25 × 50 − 9.524 = 12.50 − 9.524 = $2.98

Example 1 (cont.) — Put Pricing via Put–Call Parity

Using c₀=$2.98 | S₀=$50 | X=$55 | r=5% | T=1yr
p₀ = c₀ − S₀ + X(1+r)^−T
= 2.98 − 50 + 55 × (1.05)^−1
= 2.98 − 50 + 52.38
= $5.36

With ±40% volatility: c₀=$8.04
p₀ = 8.04 − 50 + 52.38 = $10.42

Higher volatility → BOTH call AND put values increase significantly.

Example 2 — Risk-Neutral Pricing (Hightest Capital revisited)

S₀=$50 | X=$55 | r=5% | T=1yr | S₁ᵘ=$60 | S₁ᵈ=$40 | Rᵘ=1.20 | Rᵈ=0.80
STEP 1: Compute risk-neutral probability
π = (1 + r − Rᵈ) / (Rᵘ − Rᵈ)
= (1.05 − 0.80) / (1.20 − 0.80)
= 0.25 / 0.40
= 0.625
(1−π) = 0.375

STEP 2: Price the CALL
c₁ᵘ = $5, c₁ᵈ = $0
c₀ = [0.625 × 5 + 0.375 × 0] / 1.05
= 3.125 / 1.05 = $2.98 ✓ matches hedge portfolio

STEP 3: Price the PUT (same π!)
p₁ᵘ = Max(0, 55−60) = $0 | p₁ᵈ = Max(0, 55−40) = $15
p₀ = [0.625 × 0 + 0.375 × 15] / 1.05
= 5.625 / 1.05 = $5.36 ✓ matches put–call parity

STEP 4: Confirm put–call parity
S₀ + p₀ = 50 + 5.36 = $55.36
c₀ + X(1+r)^−T = 2.98 + 55/1.05 = 2.98 + 52.38 = $55.36 ✓
★

Exam Checklist & Quick Reference Summary

  • ✅ Why binomial? Options require a price model because payoffs are asymmetric and depend on whether S_T > X. Forward commitments don't need a model (symmetric payoff). The binomial model is the simplest valid model: two possible outcomes (up or down).
  • ✅ Binomial model setup: S₁ᵘ = Rᵘ×S₀ (up state); S₁ᵈ = Rᵈ×S₀ (down state). Rᵘ > 1, Rᵈ < 1. The spread (S₁ᵘ − S₁ᵈ) represents volatility. Larger spread = higher volatility = higher option values for both calls and puts.
  • ✅ At T, option values are KNOWN at t=0: c₁ᵘ = Max(0, S₁ᵘ−X); c₁ᵈ = Max(0, S₁ᵈ−X). The only unknown is c₀.
  • ✅ Hedge portfolio structure: Sell 1 call option (−c₀) + Buy h* units of underlying (+h*×S₀). V₀ = h*×S₀ − c₀. Choose h* so that V₁ᵘ = V₁ᵈ → risk-free portfolio.
  • ✅ Hedge ratio formula (h*): h* = (c₁ᵘ − c₁ᵈ) / (S₁ᵘ − S₁ᵈ). All inputs known at t=0. This is the option's delta (Δ). For calls: 0 < h* < 1. For puts: h* is negative (short underlying needed).
  • ✅ No-arbitrage pricing: Risk-free portfolio must earn r. So V₁ = V₀×(1+r) → V₀ = V₁/(1+r). Then c₀ = h*×S₀ − V₀ = h*×S₀ − V₁×(1+r)^−1.
  • ✅ Hedge portfolio method — 5 steps: (1) Compute c₁ᵘ and c₁ᵈ; (2) Compute h*; (3) Compute V₁; (4) Discount: V₀ = V₁/(1+r); (5) Solve: c₀ = h*×S₀ − V₀.
  • ✅ Risk-neutral probability formula (Eq. 7): π = (1+r − Rᵈ) / (Rᵘ − Rᵈ). This is NOT the real-world probability q. It's the probability that makes the underlying earn the risk-free rate in expectation.
  • ✅ Risk-neutral pricing formula: c₀ = [π×c₁ᵘ + (1−π)×c₁ᵈ] / (1+r)^T. Works for ANY European derivative — same formula for puts (just change payoffs to Max(0, X−S_T)).
  • ✅ π is a property of the UNDERLYING, not the option. Once computed, use the same π to price any call or put on that underlying with that volatility/rate assumption. Changing X does not change π.
  • ✅ Real-world probabilities (q) do NOT affect option price. Being more optimistic (higher q for up move) does not change c₀. Only Rᵘ, Rᵈ, and r determine the no-arbitrage price. The hedge position offsets all directional exposure.
  • ✅ Higher volatility (wider Rᵘ−Rᵈ spread) → higher option value. Both call and put values increase with volatility. This is consistent with the factor analysis in Module 8 (σ ↑ → both call and put ↑).
  • ✅ No-arbitrage constraint: Rᵈ < (1+r) < Rᵘ must hold, otherwise there's a riskless arbitrage. This ensures π is between 0 and 1.
  • ✅ Put option via binomial: Same formula; p₁ᵘ = Max(0, X−S₁ᵘ); p₁ᵈ = Max(0, X−S₁ᵈ). Use same π. OR use put–call parity after computing c₀: p₀ = c₀ − S₀ + X(1+r)^−T.
  • ✅ Put–call parity confirmation: Both c₀ and p₀ from the binomial model must satisfy S₀ + p₀ = c₀ + X(1+r)^−T. If they don't, something went wrong.
  • ✅ Two methods, same result: Hedge portfolio method ≡ Risk-neutral pricing method — they are mathematically equivalent. The risk-neutral method is faster and generalises better.
  • ✅ Extension: The one-period binomial model extends to multiple periods (binomial tree), each period using the same h* recalculation. In the limit (infinite periods), it converges to the Black-Scholes-Merton model.
★ COMPLETE QUICK REFERENCE — MODULE 10
═══ BINOMIAL MODEL SETUP ═══
S₁ᵘ = Rᵘ × S₀ (up state) Rᵘ > 1
S₁ᵈ = Rᵈ × S₀ (down state) Rᵈ < 1
No-arbitrage: Rᵈ < (1+r) < Rᵘ

═══ OPTION PAYOFFS AT T (known at t=0) ═══
Call: c₁ᵘ = Max(0, S₁ᵘ−X) c₁ᵈ = Max(0, S₁ᵈ−X)
Put: p₁ᵘ = Max(0, X−S₁ᵘ) p₁ᵈ = Max(0, X−S₁ᵈ)

═══ HEDGE RATIO ═══
h* = (c₁ᵘ − c₁ᵈ) / (S₁ᵘ − S₁ᵈ) [call; positive]
h* = (p₁ᵘ − p₁ᵈ) / (S₁ᵘ − S₁ᵈ) [put; negative]

═══ HEDGE PORTFOLIO METHOD ═══
V₁ = h*×S₁ᵘ − c₁ᵘ = h*×S₁ᵈ − c₁ᵈ (must be equal)
V₀ = V₁ / (1+r)
c₀ = h*×S₀ − V₀

═══ RISK-NEUTRAL PRICING ═══
π = (1 + r − Rᵈ) / (Rᵘ − Rᵈ) [Eq. 7]
c₀ = [π×c₁ᵘ + (1−π)×c₁ᵈ] / (1+r)^T
p₀ = [π×p₁ᵘ + (1−π)×p₁ᵈ] / (1+r)^T
Same π for ALL options on same underlying

═══ KEY FACTS ═══
Real-world q → does NOT affect option price
Volatility ↑ → BOTH call AND put values ↑
Both methods give identical results
Confirm: S₀ + p₀ = c₀ + X(1+r)^−T (put–call parity)
Memorize the 5-step hedge portfolio method and the 3-step risk-neutral method. The exam may give you partial information and ask you to solve for the missing piece — being fluent in both methods ensures you can approach from any angle.

The hedge ratio h* = delta Δ. This bridges Module 8 (where delta was introduced conceptually as the sensitivity of option price to underlying price) with Module 10 (where it's derived precisely as (c₁ᵘ − c₁ᵈ)/(S₁ᵘ − S₁ᵈ)).

Risk-neutral pricing is not a trick. It's a consequence of no-arbitrage: in a world where all investors can hedge perfectly, everyone must agree on the option price regardless of their risk preferences or views on the probability of up/down moves.
