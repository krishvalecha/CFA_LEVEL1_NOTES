01

Key Definitions — At a Glance

Random Variable (X)
A quantity whose outcome is uncertain. Takes one of several possible values, each with an associated probability. Examples: stock return, EPS, portfolio value.
Expected Value — E(X)
The probability-weighted average of all possible outcomes of a random variable. Our best forecast of the variable's future value. Looks forward (forecast) or to the population mean — NOT the same as the historical/sample mean.
Variance — σ²(X)
The probability-weighted average of squared deviations from the expected value. Measures dispersion of outcomes around E(X). Always ≥ 0. Units: squared units of X (e.g., percent²).
Standard Deviation — σ(X)
The positive square root of variance. Measures dispersion in the SAME units as X. Preferred over variance for interpretation (e.g., if X is return in %, σ is also in %).
Conditional Probability — P(A | B)
The probability of event A given that event B has occurred. Formula: P(A | B) = P(AB) / P(B), where P(B) ≠ 0. The "|" symbol means "given that."
Conditional Expected Value — E(X | S)
The expected value of X given a specific scenario or event S. Updates the expected value to reflect the occurrence of S.
Mutually Exclusive & Exhaustive Events
Mutually exclusive: only one event can occur at a time. Exhaustive: together they cover ALL possible outcomes (probabilities sum to 1). Required for applying the total probability rule and Bayes' formula.
Prior Probability vs. Posterior Probability
Prior probability: initial probability before receiving new information. Posterior probability: updated probability AFTER incorporating new information (output of Bayes' formula).
⚠️ Exam Trap — Expected Value vs. Sample Mean

Expected value E(X): probability-weighted; forward-looking forecast; uses population probabilities.
Sample mean: equally-weighted average of past observations; backward-looking; historical.
These are fundamentally different concepts. Do not confuse them on the exam.

02

Expected Value — E(X)

★ Expected Value Formula
E(X) = P(X₁)·X₁ + P(X₂)·X₂ + ... + P(Xₙ)·Xₙ

E(X) = Σ P(Xᵢ) · Xᵢ  (summed from i = 1 to n)
Where Xᵢ = each possible outcome | P(Xᵢ) = probability of that outcome
All probabilities must sum to 1: Σ P(Xᵢ) = 1
Units: Same units as X (e.g., if X is stock return in %, E(X) is in %)
📊 Worked Example — Expected Stock Return

A stock has three possible annual returns depending on economic conditions:

Scenario Probability Return
─────────────────────────────────
Strong growth P = 0.30 R = 20%
Moderate growth P = 0.50 R = 10%
Recession P = 0.20 R = -5%
─────────────────────────────────
Sum of P = 1.00 ✓ (must verify this first)

E(R) = 0.30(20%) + 0.50(10%) + 0.20(-5%)
= 6.0% + 5.0% + (-1.0%)
= 10.0%
→ Expected return = 10.0%
💡 Investment Application of E(X)

Expected value is used to: estimate future stock/portfolio returns, forecast EPS and other financial variables, compare investment alternatives on a risk-adjusted basis, and set price targets. It is a probability-weighted forecast — not a guarantee. Individual outcomes will differ from E(X); hence we also measure risk via variance.

03

Variance & Standard Deviation

★ Variance Formula
σ²(X) = E[X − E(X)]²

σ²(X) = P(X₁)·[X₁ − E(X)]² + P(X₂)·[X₂ − E(X)]² + ... + P(Xₙ)·[Xₙ − E(X)]²

σ²(X) = Σ P(Xᵢ) · [Xᵢ − E(X)]²  (summed from i = 1 to n)
Notations: σ²(X) and Var(X) are interchangeable.
Units: squared units of X (e.g., percent² if X is a return)
Always σ²(X) ≥ 0. If σ² = 0 → no risk, outcome is certain.
★ Standard Deviation Formula
σ(X) = √σ²(X) = √Var(X)
Standard deviation is always the POSITIVE square root of variance.
Units: same as X — so it IS directly comparable to the outcomes and E(X).
This is why σ is preferred over σ² for interpretation and communication.
📊 Worked Example — Variance & Std Dev (continuing prior example)

Using the same stock: E(R) = 10.0%

Scenario P R [R − E(R)] [R − E(R)]² P × [R−E(R)]²
──────────────────────────────────────────────────────────────────────
Strong growth 0.30 20% 20−10 = 10% 100%² 0.30 × 100 = 30.0
Moderate 0.50 10% 10−10 = 0% 0%² 0.50 × 0 = 0.0
Recession 0.20 -5% -5−10 = -15% 225%² 0.20 × 225 = 45.0
──────────────────────────────────────────────────────────────────────
σ²(R) = 30.0 + 0.0 + 45.0 = 75.0 (%²)

σ(R) = √75.0 = 8.66%
→ Variance = 75 (%²) | Standard Deviation = 8.66%
Variance — Key Properties
  • Always ≥ 0 (sum of squared terms)
  • = 0 means no risk; outcome is certain
  • Greater variance = greater dispersion = more risk
  • Units are SQUARED units of X (not directly comparable to X)
  • σ²(X) and Var(X) are the same thing
Standard Deviation — Key Properties
  • Always the POSITIVE square root of variance
  • Same units as X → directly interpretable
  • Also called "volatility" in finance
  • Preferred measure for communicating risk
  • Has conditional version: conditional std dev given scenario S
💡 Calculation Shortcut — Step-by-Step Order

Step 1: Calculate E(X) = Σ P(Xᵢ)·Xᵢ
Step 2: For each outcome, compute the deviation: [Xᵢ − E(X)]
Step 3: Square each deviation: [Xᵢ − E(X)]²
Step 4: Weight each squared deviation by its probability: P(Xᵢ) × [Xᵢ − E(X)]²
Step 5: Sum all weighted squared deviations → σ²(X)
Step 6: Take the positive square root → σ(X)

Always verify probabilities sum to 1 before starting.

04

Conditional Probability & Conditional Expected Value

★ Conditional Probability
P(A | B) = P(AB) / P(B),  where P(B) ≠ 0

Read as: "Probability of A GIVEN THAT B has occurred"

Rearranged (Multiplication Rule):
P(AB) = P(A | B) × P(B) = P(B | A) × P(A)
P(AB) = joint probability of both A and B occurring.
Conditional probability updates the probability of A once we know B has occurred.
The denominator P(B) restricts the sample space to only outcomes where B occurred.
★ Conditional Expected Value
E(X | S) = P(X₁ | S)·X₁ + P(X₂ | S)·X₂ + ... + P(Xₙ | S)·Xₙ
E(X | S) = expected value of X given scenario S has occurred.
Uses conditional probabilities P(Xᵢ | S) instead of unconditional P(Xᵢ).
Each scenario S also has its own conditional variance σ²(X | S) and conditional standard deviation σ(X | S).

Why conditional expected values matter in investments: When we refine our expectations or forecasts based on new information or events, we are computing conditional expected values. For example:

  • E(stock return | recession) vs. E(stock return | expansion)
  • E(EPS | management beats guidance) vs. E(EPS | management misses)
  • E(bond return | rate hike) vs. E(bond return | rate cut)

Conditional expected values are the building blocks of probability trees and the total probability rule.

05

Probability Trees

A probability tree is a visual diagram that maps out the results of two or more events in sequence. Each branch shows a possible outcome and its probability. The tree makes it easy to calculate joint probabilities (probability of a sequence of events) and is the visual representation of the total probability rule.

Key rule: The probabilities on branches leaving any single node must sum to 1. Joint probability of any path = product of all probabilities along that path.

Start
  │
  ├──[P(Good Economy) = 0.60]──→ Good Economy
  │                                    │
  │                                    ├──[P(Beat EPS | Good) = 0.80]──→ Beat EPS  Joint P = 0.60 × 0.80 = 0.48
  │                                    │
  │                                    └──[P(Miss EPS | Good) = 0.20]──→ Miss EPS  Joint P = 0.60 × 0.20 = 0.12
  │
  └──[P(Bad Economy) = 0.40]──→ Bad Economy
                                       │
                                       ├──[P(Beat EPS | Bad) = 0.30]──→ Beat EPS  Joint P = 0.40 × 0.30 = 0.12
                                       │
                                       └──[P(Miss EPS | Bad) = 0.70]──→ Miss EPS  Joint P = 0.40 × 0.70 = 0.28
                                                                                                             ──────
                                                              Total (all joint probs must sum to 1):         1.00 ✓
💡 Reading a Probability Tree — Rules

Rule 1: At any node, all branch probabilities must sum to 1.
Rule 2: Joint probability of any complete path = multiply all probabilities along that path.
Rule 3: All joint probabilities across all complete paths must sum to 1.
Rule 4: Probabilities on later branches are CONDITIONAL (given the earlier event happened).
Rule 5: Unconditional probability of any event = sum of all joint probabilities leading to that event.

⚠️ Exam Trap — Confusing Unconditional and Conditional Probabilities in Trees

In a two-stage tree, the second-stage probabilities are CONDITIONAL on the first-stage outcome — they are NOT unconditional. For example, P(Beat EPS | Good Economy) = 0.80 ≠ P(Beat EPS) unconditionally. The unconditional P(Beat EPS) requires the total probability rule.

06

Total Probability Rule — For Probabilities & Expected Values

★ Total Probability Rule — For Probabilities
P(A) = P(A | S₁)·P(S₁) + P(A | S₂)·P(S₂) + ... + P(A | Sₙ)·P(Sₙ)

Simple 2-scenario version:
P(A) = P(A | S)·P(S) + P(A | Sᶜ)·P(Sᶜ)
S₁, S₂, ..., Sₙ must be MUTUALLY EXCLUSIVE and EXHAUSTIVE scenarios.
Sᶜ = complement of S (i.e., "S does not occur"), so P(S) + P(Sᶜ) = 1.
This is how you find the unconditional probability of A from a probability tree.
★ Total Probability Rule — For Expected Values
E(X) = E(X | S₁)·P(S₁) + E(X | S₂)·P(S₂) + ... + E(X | Sₙ)·P(Sₙ)

Simple 2-scenario version:
E(X) = E(X | S)·P(S) + E(X | Sᶜ)·P(Sᶜ)
The unconditional expected value equals the probability-weighted average of the CONDITIONAL expected values.
This is the "total probability rule for expected value" — the expected value version of the probability rule above.
Used in probability trees: compute E(X | each scenario) first, then weight by scenario probabilities.
📊 Worked Example — Total Probability Rule for Expected Return

Using the tree above. Given:

E(R | Good Economy) = 15%, P(Good Economy) = 0.60
E(R | Bad Economy) = -5%, P(Bad Economy) = 0.40

E(R) = E(R | Good) × P(Good) + E(R | Bad) × P(Bad)
= 15% × 0.60 + (-5%) × 0.40
= 9.0% + (-2.0%)
= 7.0%
→ Unconditional Expected Return = 7.0%
💡 The Connection: Total Probability Rule + Probability Tree

A probability tree is just the visual representation of the total probability rule. Reading across all paths of a tree and summing joint-probability-weighted outcomes gives you the unconditional expected value. The tree makes it easy to verify that you haven't missed any scenario (all joint probabilities sum to 1).

07

Bayes' Formula — Updating Probabilities

What Bayes' formula does: Updates a prior probability using new information to produce a posterior probability. It "reverses the given-that" — instead of asking P(Information | Event), it answers P(Event | Information).

Also called: inverse probability (because it reverses the conditional direction). Used whenever new observations lead you to update your beliefs about underlying causes.

★ Bayes' Formula — Three Equivalent Forms
Form 1 — Words:
Updated P(Event | Info) = [P(Info | Event) / P(Info)] × P(Event)

Form 2 — Notation:
P(Event | Information) = [P(Information | Event) / P(Information)] × P(Event)

Form 3 — With Total Probability Rule in denominator:
P(Event | Info) = P(Info | Event) × P(Event)
                   ────────────────────────────────────────────
                   Σ [P(Info | Eventᵢ) × P(Eventᵢ)] for all i
Numerator = joint probability of BOTH the information AND the event occurring.
Denominator = unconditional probability of the information (total probability rule).
The ratio [P(Info | Event) / P(Info)] is the "updating factor" — scales the prior probability up or down.
If ratio > 1 → posterior > prior (new info makes the event more likely).
If ratio < 1 → posterior < prior (new info makes the event less likely).

Step-by-Step Bayes' Formula Process

1
State the prior probabilities for each mutually exclusive and exhaustive event/scenario (these must sum to 1).
2
State the likelihoods: P(New Information | each event/scenario). These are the conditional probabilities of observing the new information, given each scenario.
3
Calculate the unconditional probability of the new information using the total probability rule: P(Info) = Σ P(Info | Eventᵢ) × P(Eventᵢ). This is the denominator.
4
Apply Bayes' formula for the event of interest: P(Event | Info) = [P(Info | Event) / P(Info)] × P(Event). This gives the posterior probability.
5
Verify: All posterior probabilities across all mutually exclusive events must sum to 1. This is a useful sanity check.
💡 Interpreting the Updating Factor

The ratio P(Info | Event) / P(Info) is the key to Bayes' formula:

— If the new information is more likely given the event than in general → ratio > 1 → posterior > prior (event becomes more probable).
— If the new information is less likely given the event than in general → ratio < 1 → posterior < prior (event becomes less probable).
— If the new information is equally likely regardless of the event → ratio = 1 → no update (information is irrelevant to this event).

⚠️ Critical Exam Points — Bayes' Formula

1. Likelihoods ≠ probabilities of scenarios. P(Info | Event) is the probability of seeing the information IF the event occurred — not the probability of the event itself.

2. The denominator P(Info) is the total (unconditional) probability of the new information — calculated using the total probability rule across ALL scenarios, not just the one you're interested in.

3. Always work with mutually exclusive and exhaustive scenarios. If scenarios don't cover all possibilities, the denominator will be wrong.

4. Bayesian updating is continuous — today's posterior becomes tomorrow's prior when new information arrives again.

08

Fully Worked Examples

Example 1 — Tech vs. Non-Tech (Frequency Approach)

📊 Setup: 500 firms — 100 Tech, 400 Non-Tech
Returns: Tech: 60 firms R > 10%, 40 firms R ≤ 10% Non-Tech: 100 firms R > 10%, 300 firms R ≤ 10% Total R > 10%: 60 + 100 = 160 firms

From frequencies:
P(Tech) = 100/500 = 0.20
P(Non-Tech) = 400/500 = 0.80
P(R > 10% | Tech) = 60/100 = 0.60
P(R > 10% | Non-Tech) = 100/400 = 0.25
P(R > 10%) = 160/500 = 0.32 ← verify with total prob rule below

Total Probability Rule check:
P(R > 10%) = 0.60 × 0.20 + 0.25 × 0.80 = 0.12 + 0.20 = 0.32 ✓

Bayes' Formula — P(Tech | R > 10%) = ?
= [P(R > 10% | Tech) / P(R > 10%)] × P(Tech)
= [0.60 / 0.32] × 0.20
= 1.875 × 0.20
= 0.375
→ P(Tech | R > 10%) = 37.5%

Intuition: Although only 20% of all firms are tech firms, among high-returning firms, 37.5% are tech firms. The new information (high return) updates the probability of being a tech firm upward from 20% to 37.5%. Verify: 60 out of 160 high-return firms are tech = 60/160 = 0.375 ✓

Example 2 — DriveMed EPS (Full Bayes' with 3 Scenarios)

📊 Setup: DriveMed announces factory capacity expansion

Step 1 — Prior probabilities:

P(EPS exceeded consensus) = 0.45
P(EPS met consensus) = 0.30
P(EPS fell short) = 0.25
Sum = 1.00 ✓

Step 2 — Likelihoods (P(Expands | each EPS outcome)):

P(Expands | EPS exceeded) = 0.75 ← high likelihood (strong sales → expansion)
P(Expands | EPS met) = 0.20 ← moderate
P(Expands | EPS fell short) = 0.05 ← low (poor results → unlikely to expand)

Step 3 — Unconditional P(Expands) via total probability rule:

P(Expands) = 0.75(0.45) + 0.20(0.30) + 0.05(0.25)
= 0.3375 + 0.0600 + 0.0125
= 0.4100 (= 41%)

Step 4 — Apply Bayes' formula for "EPS exceeded":

P(EPS exceeded | Expands) = [P(Expands | EPS exceeded) / P(Expands)] × P(EPS exceeded)
= [0.75 / 0.41] × 0.45
= 1.8293 × 0.45
= 0.8232 (≈ 82.3%)
→ Posterior P(EPS exceeded | Expands) ≈ 82.3% (vs. prior of 45%)

The expansion announcement is strong evidence of a good EPS result — the probability of EPS exceeding consensus jumped from 45% to 82.3%. The ratio 1.829 > 1 confirms the new information makes this event MORE likely.

Step 5 — Verify: compute all three posterior probabilities and check they sum to 1:

P(EPS exceeded | Expands) = [0.75/0.41] × 0.45 = 0.8232
P(EPS met | Expands) = [0.20/0.41] × 0.30 = 0.1463
P(EPS fell short | Expands)= [0.05/0.41] × 0.25 = 0.0305
─────────────────────────────────────────────────────
Sum = 0.8232 + 0.1463 + 0.0305 = 1.0000 ✓
EPS Scenario Prior P Likelihood P(Expands | Scenario) Joint P (Prior × Likelihood) Posterior P
EPS Exceeded Consensus 0.45 0.75 0.45 × 0.75 = 0.3375 0.3375 / 0.41 = 0.823
EPS Met Consensus 0.30 0.20 0.30 × 0.20 = 0.0600 0.0600 / 0.41 = 0.146
EPS Fell Short 0.25 0.05 0.25 × 0.05 = 0.0125 0.0125 / 0.41 = 0.030
Total 1.00 — P(Expands) = 0.41 1.000 ✓
💡 Tabular Method for Bayes (Exam Efficiency Tip)

The table above is the most exam-efficient way to solve Bayes' problems with 3+ scenarios. Columns: (1) Prior, (2) Likelihood, (3) Joint = Prior × Likelihood, (4) Posterior = Joint / Sum of Joints. Sum of the Joint column = unconditional P(Information). Divide each joint by this sum to get the posterior. All posteriors must sum to 1.

★

Exam Checklist — Must-Know Points

  • ✅ E(X) = Σ P(Xᵢ) · Xᵢ — probability-weighted average of all outcomes. It is a FORWARD-LOOKING forecast, NOT the historical sample mean.
  • ✅ σ²(X) = Σ P(Xᵢ) · [Xᵢ − E(X)]² — always ≥ 0. Units are SQUARED units of X. If σ² = 0, the outcome is certain (no risk).
  • ✅ σ(X) = √σ²(X) — always the positive square root. Same units as X. Preferred for interpretation and communication over variance.
  • ✅ Conditional probability P(A | B) = P(AB) / P(B) — probability of A given B occurred. Restricts sample space to outcomes where B occurred.
  • ✅ Multiplication rule: P(AB) = P(A | B) × P(B) — used to find joint probabilities from conditional ones.
  • ✅ Conditional E(X | S) = Σ P(Xᵢ | S) · Xᵢ — uses conditional probabilities; has its own conditional variance and standard deviation.
  • ✅ Total probability rule (probabilities): P(A) = Σ P(A | Sᵢ) · P(Sᵢ) over all mutually exclusive and exhaustive scenarios.
  • ✅ Total probability rule (expected values): E(X) = Σ E(X | Sᵢ) · P(Sᵢ) — unconditional E(X) is the weighted average of conditional expected values.
  • ✅ Probability tree: visual tool for 2+ sequential events. Branch probabilities leaving any node sum to 1. Joint probability of any path = product of all branch probabilities along that path. All joint probabilities across all paths sum to 1.
  • ✅ Scenarios for trees/total probability rule must be mutually exclusive AND exhaustive. If not exhaustive, probabilities won't sum to 1 and the rules don't apply correctly.
  • ✅ Bayes' formula: P(Event | Info) = [P(Info | Event) / P(Info)] × P(Event). Updates prior probability to posterior probability using new information.
  • ✅ Bayes' formula reverses the conditional: turns P(Info | Event) into P(Event | Info). Also called "inverse probability."
  • ✅ Prior probability = before new information. Posterior probability = after incorporating new information (the answer from Bayes').
  • ✅ Likelihood = P(new information | scenario). NOT the probability of the scenario itself. Required for Bayes' updating.
  • ✅ Denominator in Bayes' = unconditional P(Information), computed using the total probability rule across all scenarios. It normalizes the posteriors so they sum to 1.
  • ✅ Updating factor = P(Info | Event) / P(Info). If > 1 → posterior > prior. If < 1 → posterior < prior. If = 1 → no update (information irrelevant).
  • ✅ All posterior probabilities from Bayes' must sum to 1 — use this as your verification check in exam questions.
  • ✅ Bayesian updating is continuous — today's posterior becomes tomorrow's prior when new information arrives. Beliefs are never fixed.
  • ✅ Tabular method for multi-scenario Bayes': Columns = Prior | Likelihood | Joint (Prior × Likelihood) | Posterior (Joint / Sum of Joints). Fastest exam approach.

P(Event | Info) = [P(Info | Event) / P(Info)] × P(Event)

Where: P(Info) = Σ P(Info | Eventᵢ) × P(Eventᵢ)  ← total prob rule

═══ TABULAR BAYES SETUP ═══
[1] Prior P(Eventᵢ)
[2] Likelihood P(Info | Eventᵢ)
[3] Joint = [1] × [2]
[4] P(Info) = Σ [3] (sum of all joints = denominator)
[5] Posterior = [3] / [4]  ← final answer; all must sum to 1

Variance = 0: No dispersion, no risk, outcome is certain.

Posterior probabilities must always sum to 1 — instant exam sanity check.
