01

Essential Definitions

Null Hypothesis (H₀)
The statement considered to be true unless the sample gives convincing evidence otherwise. It is what we want to reject. Always stated in terms of population parameters. Typically represents "no effect," "no difference," or a specific hypothesized value.
Alternative Hypothesis (Hₐ)
The statement we accept if we reject H₀. Must be mutually exclusive and collectively exhaustive with H₀ — together they cover all possible values. Can be one-sided (left or right tail) or two-sided.
Test Statistic
A value calculated from the sample that is compared to a critical value to decide whether to reject H₀. The choice of test statistic depends on what is being tested and what is known/unknown.
Critical Value
The boundary value(s) from the test statistic's distribution, determined by the significance level α and whether the test is one- or two-tailed. If the calculated test statistic exceeds the critical value, we reject H₀.
Level of Significance (α)
The probability of making a Type I error (rejecting a true H₀). Set by the analyst before the test. Common values: 1%, 5%, 10%. Confidence level = 1 − α.
p-value
The smallest level of significance at which the null hypothesis can be rejected. It is the probability of obtaining a test statistic as extreme as (or more extreme than) the observed value, assuming H₀ is true. Reject H₀ if p-value < α.
Statistically Significant
A result is statistically significant when the calculated test statistic is more extreme than the critical value — i.e., we have sufficient evidence to reject H₀ at the chosen significance level.
Power of a Test (1 − β)
The probability of correctly rejecting a false H₀. Power = 1 − P(Type II error). Higher power is better. Increases with: larger sample size, higher significance level α, larger true difference from H₀.
💡 H₀ vs. Hₐ — How to Set Them Up

The null hypothesis always contains the equality (=, ≤, ≥). The alternative is what you are trying to prove. One-sided test: Hₐ is ">" or "<". Two-sided test: Hₐ is "≠". Always write hypotheses in terms of population parameters (µ, σ²), not sample statistics (X̄, s²).

02

The Six-Step Hypothesis Testing Process

1
State the hypotheses. Formulate H₀ and Hₐ in terms of population parameters. They must be mutually exclusive and collectively exhaustive. Decide if one-tailed or two-tailed based on the research question.
2
Identify the appropriate test statistic and its probability distribution. Choose the correct test statistic (t, z, χ², F) based on what you're testing and the data properties (known/unknown σ, sample sizes, independence of samples).
3
Specify the level of significance (α). Choose α before collecting data (typically 1%, 5%, or 10%). This reflects how much evidence you need to reject H₀ and the cost/consequence of a Type I error.
4
State the decision rule. Determine the critical value(s) from the distribution at the chosen α and degrees of freedom. Define the rejection region: if calculated test statistic falls in rejection region → reject H₀.
5
Collect the data and calculate the test statistic. Use the sample data to compute the numerical value of the test statistic using the appropriate formula.
6
Make a decision. Compare the calculated test statistic to the critical value(s). Reject or fail to reject H₀. State the conclusion in the context of the original question. Note: "fail to reject" ≠ "accept" H₀.
⚠️ Exam Trap: "Fail to Reject" ≠ "Accept" H₀

We never "accept" the null hypothesis. We either reject it or fail to reject it. Failing to reject H₀ means the data do not provide sufficient evidence against it — not that H₀ is proven true. This language distinction matters on the exam.

One-Tailed vs. Two-Tailed Tests

Two-Tailed Test (Hₐ: µ ≠ µ₀)
  • Tests if parameter differs in either direction from H₀.
  • Rejection region split equally: α/2 in each tail.
  • Critical values: ±t(α/2, df) or ±z(α/2).
  • Example: "Is the fund's return different from 1.1%?"
  • More conservative — harder to reject H₀ for a given α.
One-Tailed Test (Hₐ: µ > µ₀ or µ < µ₀)
  • Tests if parameter is greater than OR less than H₀ value.
  • All α concentrated in one tail.
  • Critical value: t(α, df) or z(α) — only one boundary.
  • Example: "Is the fund's variance less than 16%²?"
  • More powerful for detecting a difference in the specified direction.

Decision Rule: Test Statistic vs. Critical Value

Reject H₀ if: |calculated test statistic| > critical value (two-tailed), OR calculated test statistic > +critical value (right-tail), OR calculated test statistic < −critical value (left-tail).

Alternative via p-value: Reject H₀ if p-value < α. The p-value is the area in the tail(s) beyond the calculated test statistic. This gives the same conclusion as the critical value approach.

💡 Confidence Interval Approach to Hypothesis Testing

A two-tailed hypothesis test at significance level α is equivalent to constructing a (1−α) confidence interval. Reject H₀ if the hypothesized value falls outside the confidence interval. CI = X̄ ± critical value × (s/√n). This is a faster approach on the exam when the CI is already calculated.

Type I Error (α) — False Positive
  • Rejecting H₀ when it is actually TRUE.
  • Probability = α = significance level.
  • Controlled directly by the analyst's choice of α.
  • Lower α → less likely to make Type I error → but more likely to make Type II error.
  • Investment analogy: Concluding a manager has skill when they don't.
Type II Error (β) — False Negative
  • Failing to reject H₀ when it is actually FALSE.
  • Probability = β. Not directly controlled by analyst.
  • Decreases with: larger n, larger α, larger true effect size.
  • Cannot simultaneously minimize both α and β without increasing n.
  • Investment analogy: Concluding a manager has no skill when they do.
Error Probabilities and Power — Complete Summary
α = P(Type I error) = P(Reject H₀ | H₀ is true) = significance level
β = P(Type II error) = P(Fail to reject H₀ | H₀ is false)
Power = 1 − β = P(Reject H₀ | H₀ is false) = probability of correct rejection
Confidence level = 1 − α = probability of correct retention

Trade-off: ↓α → ↑β (and ↓Power) | ↑α → ↓β (and ↑Power)
Only way to reduce BOTH α and β simultaneously: INCREASE sample size n.
Power increases with: (1) larger sample size n, (2) larger significance level α, (3) larger true difference between hypothesized and actual parameter value, (4) lower population variance (data less noisy).
🧠 Memory Trick: Type I vs. Type II

Type I = "Crying wolf" — you say the wolf is there (reject H₀) but it isn't (H₀ is true). A false alarm. α controls this. Type II = "Missing the wolf" — you say there's no wolf (fail to reject H₀) but there is one (H₀ is false). A missed detection. β controls this. You'd rather cry wolf occasionally than miss the actual wolf — so in safety-critical contexts, we prefer controlling Type I error strictly (very small α), accepting higher Type II risk.

💡 Which Test to Use — Decision Tree

Testing a MEAN? → t-test (σ unknown, nearly always). Testing a VARIANCE? → chi-square. Comparing TWO VARIANCES? → F-test. Comparing TWO MEANS (independent)? → t-test (pooled or separate variances). Comparing TWO MEANS (dependent/paired)? → paired t-test (test of mean differences). Testing CORRELATION? → t-test with n−2 df. Categorical data independence? → chi-square.

05

Test of a Single Mean (t-test)

t-statistic for Single Mean (σ Unknown — Use in Practice)
t = (X̄ − µ₀) / (s / √n)

Degrees of freedom: df = n − 1
X̄ = sample mean  |  µ₀ = hypothesized population mean  |  s = sample standard deviation  |  n = sample size.
The denominator (s/√n) is the standard error of the sample mean.
When σ IS known (very rare), use z = (X̄ − µ₀) / (σ/√n) and the normal distribution instead.
📐 Worked Example: Sendar Equity Fund — Single Mean Test

Setup: Fund's actual mean monthly return X̄ = 1.50%, s = 3.60%, n = 24 months. Pricing model predicts µ₀ = 1.10%. Test at α = 5% (two-tailed): Is actual performance different from the model prediction?

Step 1: H₀: µ = 1.1% vs. Hₐ: µ ≠ 1.1%

Step 2: Test statistic → t with df = 24 − 1 = 23

Step 3: α = 5%, two-tailed

Step 4: Critical values = ±2.069 (t at 2.5% in each tail, 23 df)

Step 5: t = (1.50 − 1.10) / (3.60 / √24)
t = 0.40 / (3.60 / 4.899)
t = 0.40 / 0.7348
t = 0.544
Step 6: |0.544| < 2.069 → FAIL TO REJECT H₀

Conclusion: Insufficient evidence that the fund's mean monthly return differs from the 1.10% pricing model prediction. The outperformance is not statistically significant at 5%.

Confidence Interval check: CI = 1.50 ± 2.069 × (3.60/√24) = 1.50 ± 1.52 = [−0.02%, 3.02%]. Since µ₀ = 1.1% falls within [−0.02%, 3.02%], we fail to reject H₀ — same conclusion.

⚠️ Statistical vs. Economic Significance

A result can be statistically significant but economically trivial, or statistically insignificant but economically meaningful. A very large sample can make tiny differences statistically significant. Always ask: "Even if statistically significant, is the effect size large enough to matter economically?" This distinction frequently appears on the exam.

06

Test of a Single Variance (Chi-Square Test)

Chi-Square (χ²) Test Statistic for Single Variance
χ² = (n − 1) × s² / σ²₀

Degrees of freedom: df = n − 1
s² = observed sample variance  |  σ²₀ = hypothesized population variance  |  n = sample size.
Assumption: the population is normally distributed. Chi-square is NOT robust to departures from normality.
The chi-square distribution is right-skewed and bounded below by 0 (variance cannot be negative). The shape depends on df.

Three possible hypothesis set-ups for variance:

  • Two-tailed: H₀: σ² = σ²₀ vs. Hₐ: σ² ≠ σ²₀ → reject if χ² < χ²lower or χ² > χ²upper
  • Right-tail (testing if variance is large): H₀: σ² ≤ σ²₀ vs. Hₐ: σ² > σ²₀ → reject if χ² > critical value
  • Left-tail (testing if variance is small): H₀: σ² ≥ σ²₀ vs. Hₐ: σ² < σ²₀ → reject if χ² < critical value
📐 Worked Example: Sendar Equity Fund — Single Variance Test

Setup: Sendar Equity Fund, n = 24, s = 3.60%, so s² = 12.96% (= 0.1296 in decimal, but here in % squared: 12.96). Test at α = 5%: Is σ² < 16 (i.e., σ < 4%)? H₀: σ² ≥ 16 vs. Hₐ: σ² < 16 (one-tailed, left tail). df = 23.

Critical value (left tail, 5% in left): χ²(0.05, 23) = 13.09051. Reject H₀ if χ² < 13.09051.

χ² = (24 − 1) × 12.96 / 16
χ² = 23 × 12.96 / 16
χ² = 298.08 / 16
χ² = 18.63 > 13.09 → FAIL TO REJECT H₀

Conclusion: Insufficient evidence that the fund's variance is less than 16 (σ < 4%). The data do not support the claim of lower volatility.

💡 Chi-Square: Key Properties to Remember

Chi-square distribution is asymmetric (right-skewed) and always positive. Critical values are NOT symmetric around zero (unlike t and z). For a left-tail test, you find the value where only α% of the distribution lies to the LEFT — this will be a small positive number. For a right-tail test, it's a large positive number. Always specify df = n−1 for single variance tests.

07

Tests Concerning Differences Between Means

When comparing means of two groups, the first decision is: Are the samples independent or dependent (paired)? This determines which test to use.

Case A: Independent Samples (Pooled t-test — Equal Variances Assumed)

t-test: Difference in Means (Independent Samples, Equal Variances)
t = [(X̄₁ − X̄₂) − (µ₁ − µ₂)₀] / √(s²p/n₁ + s²p/n₂)

Pooled variance: s²p = [(n₁−1)s²₁ + (n₂−1)s²₂] / (n₁ + n₂ − 2)

Degrees of freedom: df = n₁ + n₂ − 2
Under H₀: µ₁ − µ₂ = 0, so (µ₁ − µ₂)₀ = 0 in the numerator, simplifying to t = (X̄₁ − X̄₂) / √(s²p/n₁ + s²p/n₂).
The pooled variance s²p combines both samples' variance estimates into one, weighted by degrees of freedom. This is valid ONLY when variances are assumed equal.
Assumption: both samples from normally distributed populations; samples are independent.

Case B: Dependent (Paired) Samples — Paired Comparisons Test

t-test: Mean of Differences (Paired Comparisons)
d̄ = (1/n) × Σᵢ dᵢ where dᵢ = X₁ᵢ − X₂ᵢ (difference for each pair)

t = (d̄ − µd₀) / s_d̄

s_d̄ = s_d / √n (standard error of mean difference)

Degrees of freedom: df = n − 1
Under H₀: µd = 0 (no mean difference), so t = d̄ / s_d̄.
d̄ = sample mean of the paired differences.
s_d = sample standard deviation of the paired differences.
n = number of PAIRS (not total observations).
When to Use Paired Test
  • Samples are dependent — observations are naturally linked in pairs.
  • Same subjects measured twice (before/after): e.g., portfolio returns before and after a strategy change.
  • Matched pairs: e.g., comparing two funds over the same time periods (same economic conditions).
  • df = n − 1 where n = number of pairs.
Why Paired Test is More Powerful
  • By pairing observations (e.g., same time period), you eliminate variation caused by factors common to both samples (market conditions, interest rate environment, etc.).
  • The residual variation (the differences) is smaller, so the test can more easily detect a true difference between means.
  • More powerful than the independent-samples test for the same n pairs.
⚠️ Exam Trap: Independent vs. Dependent Samples

If the question says "same companies" or "same time periods" for both samples → dependent/paired → use paired t-test. If the two samples are drawn from different, unrelated groups → independent → use pooled t-test. The df formula is different: paired uses n−1 (n = number of pairs); independent uses n₁+n₂−2.

08

Test of Equality of Two Variances (F-test)

F-test: Ratio of Two Sample Variances
F = s²₁ / s²₂

Degrees of freedom: df₁ = n₁ − 1 (numerator), df₂ = n₂ − 1 (denominator)
Convention: Place the larger sample variance in the numerator (s²₁ ≥ s²₂) → F ≥ 1. This simplifies the test to a right-tail test only.
H₀: σ²₁ = σ²₂ (variances are equal) vs. Hₐ: σ²₁ ≠ σ²₂ (variances differ).
Reject H₀ if F > F_critical (right-tail critical value at α/2 for two-tailed, or α for one-tailed).
Assumptions: both populations are normally distributed; samples are independent and random.
Finance applications: Compare volatility of two portfolios, two funds, or the same asset in different time periods.

Reading F-distribution tables: The F-distribution is parameterized by two degrees of freedom — numerator (df₁) and denominator (df₂). Critical values are always positive (F ≥ 0). The distribution is right-skewed. Always put the larger variance in the numerator so you only need the right-tail critical value.

🧠 Summary: Which Test for Which Situation

One mean, σ unknown → t-test (df = n−1)
One variance → χ² test (df = n−1)
Two means, independent → pooled t-test (df = n₁+n₂−2)
Two means, dependent → paired t-test (df = n−1, n = pairs)
Two variances → F-test (df₁ = n₁−1, df₂ = n₂−1)
Correlation → t-test (df = n−2)

09

Parametric vs. Nonparametric Tests

Parametric Tests
  • Concerned with estimating or testing a specific population parameter (µ, σ², etc.).
  • Validity depends on specific distributional assumptions (usually normality).
  • Examples: t-test, z-test, F-test, chi-square test for variance.
  • Generally preferred when assumptions are met — more statistically powerful (better at detecting false H₀).
Nonparametric Tests
  • Either NOT concerned with a parameter, OR make minimal distributional assumptions.
  • Often work with ranks or signs rather than raw data values.
  • Require specialized statistical tables for critical values.
  • Less powerful than parametric tests when parametric assumptions are met.
  • But more reliable when parametric assumptions are violated.

Four Situations Where Nonparametric Tests Are Appropriate

1
Distributional assumptions are NOT met. Data comes from a small sample that appears to come from a markedly non-normal population. Parametric t-tests or z-tests are not valid; use nonparametric alternatives.
2
Data contains outliers. Extreme values distort parametric statistics (e.g., mean, variance) but not rank-based nonparametric statistics. Example: testing the median (robust to outliers) instead of the mean.
3
Data is in ranks or ordinal scale. Parametric tests generally require interval or ratio scale data. If observations are rankings of investment managers or analyst ratings, use nonparametric procedures.
4
Hypothesis does not concern a parameter. Example: testing whether a sample is random (runs test) — used to test the random walk theory for stock price changes. Or testing whether data follows a specific distribution.

Nonparametric Alternatives to Common Parametric Tests

Test Purpose Parametric Alternative Nonparametric Alternative
Test of a single mean t-test or z-test Wilcoxon signed-rank test
Test of difference between means (independent) t-test (pooled) Mann–Whitney U test (Wilcoxon rank sum test)
Test of mean differences (paired/dependent) Paired t-test Wilcoxon signed-rank test; Sign test
Test of randomness n/a (no parametric equivalent) Runs test (tests random walk hypothesis)
💡 When Are Both Tests Run Together?

Analysts often report both parametric and nonparametric results side by side. This lets the user assess how sensitive the conclusion is to the distributional assumptions of the parametric test. If both tests agree → conclusion is robust. If they disagree → the parametric assumptions may be questionable and further investigation is warranted.

⚠️ Nonparametric ≠ Better

Nonparametric tests are NOT automatically preferred. When parametric assumptions ARE met, the parametric test generally has more statistical power — i.e., a greater probability of correctly rejecting a false H₀. Nonparametric tests are chosen only when the conditions that make parametric tests valid are absent.

★

Exam Checklist — Must-Know Points

  • ✅ Six steps of hypothesis testing: (1) State hypotheses, (2) Identify test statistic & distribution, (3) Specify α, (4) State decision rule, (5) Calculate test statistic, (6) Make decision. Follow in order.
  • ✅ H₀ always contains the equality (=, ≤, ≥). Hₐ is what you're trying to prove. Both are in terms of population parameters, not sample statistics.
  • ✅ Never "accept" H₀. You either REJECT it or FAIL TO REJECT it. Failing to reject ≠ proving it true.
  • ✅ Type I error (α): Reject a TRUE H₀ = false positive. Controlled by choice of α. Type I = "crying wolf."
  • ✅ Type II error (β): Fail to reject a FALSE H₀ = false negative. Type II = "missing the wolf."
  • ✅ Power = 1 − β = probability of correctly rejecting a false H₀. Want power to be high. Increases with larger n, larger α, larger true effect size.
  • ✅ α and β trade-off: Decreasing α increases β (and reduces power). The ONLY way to reduce both simultaneously is to increase sample size n.
  • ✅ p-value = smallest α at which H₀ can be rejected. Reject H₀ if p-value < α. p-value is the area in the tail(s) beyond the calculated test statistic.
  • ✅ Confidence interval approach: Reject H₀ if hypothesized value falls outside the (1−α) CI. Equivalent to the two-tailed test at significance level α.
  • ✅ t-test for single mean: t = (X̄ − µ₀) / (s/√n), df = n−1. Use when σ is unknown (almost always). Use z when σ is known.
  • ✅ Chi-square for single variance: χ² = (n−1)s² / σ²₀, df = n−1. Requires normally distributed population. Distribution is right-skewed, asymmetric, always positive.
  • ✅ Pooled t-test for two independent means: t uses pooled variance s²p, df = n₁+n₂−2. Requires approximately equal population variances.
  • ✅ Paired t-test for dependent means: t = d̄ / s_d̄, df = n−1 where n = number of pairs. More powerful because it eliminates common variation between paired observations.
  • ✅ Independent vs. dependent samples: Same time periods / same entities → dependent → paired test. Unrelated groups → independent → pooled t-test.
  • ✅ F-test for two variances: F = s²₁/s²₂ with larger variance in numerator. df = (n₁−1, n₂−1). Requires normal populations, independent samples. Used to compare volatility of two assets/periods.
  • ✅ Correlation t-test: t = r√(n−2)/√(1−r²), df = n−2.
  • ✅ Statistical significance ≠ economic significance. Large n can make trivial differences significant. Always assess practical importance.
  • ✅ Parametric tests: Concern population parameters; require distributional assumptions (usually normality). Preferred when assumptions are met (more powerful).
  • ✅ Nonparametric tests: Use when: (1) distributional assumptions violated, (2) outliers present, (3) data is in ranks, (4) hypothesis is not about a parameter (e.g., runs test for randomness).
  • ✅ Key nonparametric alternatives: Wilcoxon signed-rank (single mean / paired); Mann–Whitney U (difference in means, independent); Runs test (test of randomness / random walk).
