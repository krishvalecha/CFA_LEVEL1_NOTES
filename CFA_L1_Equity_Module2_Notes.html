Index Basics — Definitions

Security Market Index
Represents a given security market, market segment, or asset class. Constructed as a portfolio of constituent securities. Value is calculated regularly using actual or estimated market prices of constituents.
Price Return Index (Price Index)
  • Reflects only prices of constituent securities
  • Does NOT include income (dividends/interest)
  • Value increases only when constituent prices rise
Total Return Index
  • Reflects prices PLUS reinvestment of all income received since inception (dividends and/or interest)
  • At inception: Total Return Index Value = Price Return Index Value
  • Over time: Total Return Index > Price Return Index by increasing amount
The Divisor
A number chosen at inception to set the index at a convenient initial value (e.g., 1,000). The index provider adjusts the divisor whenever non-price-related events occur (e.g., stock splits, constituent changes) to prevent artificial jumps or drops in the index value. Divisor adjustments preserve index continuity.
💡 Quick History — First Indexes

1884: Dow Jones Average (9 railroads + 2 industrials) introduced by Charles Dow and Edward Jones → eventually became the Dow Jones Transportation Average. May 1896: Dow Jones Industrial Average (DJIA) introduced with 12 stocks and initial value of 40.94. Today: thousands of indexes across all asset classes worldwide.

Price Return — Formulas & Calculations

★ Price Return Index Value
V_PRI = Σ(nᵢ × Pᵢ) / D
V_PRI = value of the price return index
nᵢ = number of units of constituent security i held in the index portfolio
Pᵢ = unit price of constituent security i
D = divisor (adjusted to preserve index continuity on non-price events)
N = number of constituent securities
★ Price Return of the Index (Single Period)
PR_I = (V_PRI1 − V_PRI0) / V_PRI0
PR_I = price return of the index portfolio (decimal, e.g., 12% = 0.12)
V_PRI1 = index value at end of period | V_PRI0 = index value at beginning

Also equals the weighted average of individual security price returns:
PR_I = Σ wᵢ × PRᵢ = w₁PR₁ + w₂PR₂ + ... + wₙPRₙ
★ Price Return of Each Constituent Security
PRᵢ = (Pᵢ1 − Pᵢ0) / Pᵢ0
PRᵢ = price return of constituent security i (decimal)
Pᵢ1 = price of security i at end of period | Pᵢ0 = price at beginning of period
03

Total Return — Formulas & Calculations

★ Total Return of the Index (Single Period)
TR_I = (V_PRI1 − V_PRI0 + Inc_I) / V_PRI0
TR_I = total return of the index portfolio (decimal)
Inc_I = total income (dividends and/or interest) from ALL securities in the index over the period
Note: the denominator is V_PRI0 (price return index beginning value), NOT total return index beginning value

Also equals the weighted average of individual security total returns:
TR_I = Σ wᵢ × TRᵢ = w₁TR₁ + w₂TR₂ + ... + wₙTRₙ
★ Total Return of Each Constituent Security
TRᵢ = (P1ᵢ − P0ᵢ + Incᵢ) / P0ᵢ
TRᵢ = total return of constituent security i (decimal)
P1ᵢ = price at end of period | P0ᵢ = price at beginning of period
Incᵢ = dividends and/or interest from security i over the period
⚠️ Exam Trap — Price Return vs. Total Return Index

Both versions use the same constituent securities and same weights — they differ only in whether income is included. At inception they are equal. The total return index grows faster than the price return index over time. On a CFA question, if the index pays dividends and the question asks for price return — do NOT add the income. If it asks for total return — you MUST add income to the numerator.

04

Multi-Period Index Value Calculations

Calculating index values over multiple periods requires geometric linking (chain-multiplying) of period returns. Simple addition of returns is WRONG for multi-period calculations.

★ Multi-Period Price Return Index Value
V_PRI_T = V_PRI_0 × (1 + PR_I1) × (1 + PR_I2) × ... × (1 + PR_IT)
V_PRI_0 = index value at inception | V_PRI_T = index value at time T
PR_It = price return for period t (decimal)

Example: Inception value = 1,000; Period 1 return = 5%; Period 2 return = 3%
End of Period 1: 1,000 × 1.05 = 1,050
End of Period 2: 1,000 × 1.05 × 1.03 = 1,081.50
★ Multi-Period Total Return Index Value
V_TRI_T = V_TRI_0 × (1 + TR_I1) × (1 + TR_I2) × ... × (1 + TR_IT)
V_TRI_0 = total return index value at inception
TR_It = total return for period t (includes income)

Example (same index, adding income): Period 1 total return = 6.5%; Period 2 = 5%
End of Period 1: 1,000 × 1.065 = 1,065
End of Period 2: 1,000 × 1.065 × 1.05 = 1,118.25

Compare: Price return index = 1,081.50 vs. Total return index = 1,118.25
The gap WIDENS over time as more income is reinvested.
💡 Geometric Linking — Why Not Arithmetic?

If an index returns +50% in Year 1 and −50% in Year 2, arithmetic says 0% cumulative. But the real answer: 1,000 × 1.50 × 0.50 = 750. You have LOST 25%. Always chain-multiply. This is the same principle as compound returns in any investment context.

05

Index Construction — 5 Key Decisions

Building an index is like building a portfolio. Index providers must decide:

  1. Which target market should the index represent?
  2. Which securities to select from that target market?
  3. How much weight to allocate to each security?
  4. When to rebalance the index?
  5. When to reconstitute (re-examine security selection and weighting)?

Target Market & Security Selection

The target market may be defined by:

  • Asset class: equities, fixed income, real estate, commodities, hedge funds
  • Geography: Japan, South Africa, Latin America, Europe
  • Exchange: Shanghai, Toronto, Tokyo
  • Other characteristics: economic sector, company size, investment style, duration, credit quality

Once target market is defined → determines the investment universe. Index may include all securities in target market OR a representative sample.

Fixed-Number Indexes

Number of securities stated in name. Examples: S&P 500 (500 stocks), FTSE 100 (100 stocks), DJIA (30 stocks). Selection committee may use subjective criteria (e.g., S&P 500, S&P BSE SENSEX).

Variable-Number Indexes

Number varies to reflect changes in target market or to maintain a % of market. Example: TOPIX — includes ALL stocks in Tokyo Stock Exchange First Section meeting criteria (shares outstanding, shareholders, market cap). Stocks removed if they no longer meet criteria.

Weighting Methods — Detail

💡 Mnemonic — 4 Weighting Methods: "PEMF"

Price weighting · Equal weighting · Market-cap weighting (incl. float-adjusted) · Fundamental weighting. Know the formula, advantage, disadvantage, and key example for each.

1 — Price Weighting

Price Weighting
Weight Formula
wᵢᴾ = Pᵢ / Σ Pᵢ (sum of all prices)
Initial divisor = number of securities in the index. After stock split → adjust divisor so index value is unchanged.
New divisor = Σ prices after split / index value before split
  • Key example: DJIA (30 stocks, price-weighted); Nikkei 225 (modified price-weighted)
  • Advantage: Simple to calculate and understand
  • Disadvantage: Weights are arbitrary — determined solely by price level, not economic significance. Higher-priced stocks dominate regardless of company size. A stock split causes arbitrary weight changes in ALL constituents. A $100 stock has 10× the weight of a $10 stock even if the $10 company is far larger by market cap
  • Rebalancing: NOT rebalanced — weights self-adjust with prices

2 — Equal Weighting

Equal Weighting
Weight Formula
wᵢᴱ = 1 / N
N = number of securities in the index. Initial investment is equal dollar amount in each security. Number of shares per security = (equal dollar investment) / (share price).
  • Advantage: Simple; gives small companies equal footing with large ones
  • Disadvantages: (1) Large-cap stocks are underrepresented; small-cap stocks are overrepresented relative to their actual market weight. (2) Weights drift immediately after inception as prices change → requires frequent rebalancing. (3) Weights are arbitrarily assigned by index provider (not by market)
  • Rebalancing: Requires most frequent rebalancing of all methods — continuously drifts from equal weights as prices change

3 — Market-Capitalization Weighting

Market-Cap Weighting (Value Weighting)
Market Cap Weight Formula
wᵢᴹ = (Qᵢ × Pᵢ) / Σ (Qⱼ × Pⱼ)
Qᵢ = shares outstanding of security i | Pᵢ = share price
Numerator = market cap of security i; Denominator = total market cap of all constituents
Float-Adjusted Market Cap Weight Formula
wᵢᴹ = (fᵢ × Qᵢ × Pᵢ) / Σ (fⱼ × Qⱼ × Pⱼ)
fᵢ = fraction of shares outstanding in market float (publicly available shares)
Excludes: shares held by controlling shareholders, other corporations, governments, and (for global indexes) shares unavailable to foreign investors.
Most modern market-cap indexes are float-adjusted. "Free-float-adjusted" is another term.
  • Key examples: S&P 500, MSCI World, TOPIX, Russell 3000, FTSE 100 — virtually all major modern indexes
  • Advantage: Constituent securities are held in proportion to their value in the target market → reflects actual economic weight
  • Disadvantages: (1) Overweights stocks that have risen in price (may be overvalued) and underweights stocks that have fallen (may be undervalued). (2) Momentum effect — winning stocks get larger weight over time. (3) Closely-held stocks distort weights without float adjustment
  • Rebalancing: Largely self-rebalancing — weights automatically adjust as prices change. Only adjusted manually for mergers, acquisitions, liquidations, and other corporate actions

4 — Fundamental Weighting

Fundamental Weighting
Fundamental Weight Formula
wᵢᶠ = Fᵢ / Σ Fⱼ
Fᵢ = a fundamental size measure of company i (e.g., book value, earnings, revenues, cash flow, dividends, number of employees)
Weight is based on economic fundamentals, NOT market price.
  • Key properties:
    • "Value" tilt: higher ratios of book value/earnings/dividends to market value than market-cap-weighted counterpart → favors cheap (high-yield) stocks
    • Contrarian effect: weights shift away from stocks that have risen in price (become expensive) and toward stocks that have fallen (become cheap) at rebalancing → opposite of market-cap momentum
    • Overweights high earnings yield stocks; underweights low earnings yield stocks relative to market cap weights
  • Disadvantage: More complex; requires fundamental data; may not reflect current market information; may overweight value traps
⚠️ Exam Trap — Market-Cap vs. Fundamental: Momentum vs. Contrarian

Market-cap weighted → momentum effect: rising stocks get larger weight → like buying more of what has already gone up. Fundamental weighted → contrarian effect: at rebalancing, trims winners (now more expensive) and adds to losers (now cheaper). This distinction is a common exam question. "Which weighting method has a value tilt?" → Fundamental. "Which mimics a momentum strategy?" → Market-cap.

💡 Key Insight — Same Stocks, Different Returns

From the module's example: the same 5 securities produced price returns of 3.45% (price-weighted), 10.40% (equal-weighted), and 1.49% (market-cap-weighted). The weighting method alone determines the return. On the exam, if asked why two indexes of the same market have different returns, the answer is often the weighting method.

Rebalancing & Reconstitution

Rebalancing

Definition: Adjusting the weights of existing constituent securities back to their target weights.

  • Done on regularly scheduled dates — usually quarterly
  • Necessary because prices change → weights drift from targets
  • Creates turnover in the index (sells winners, buys laggards for equal-weighted)
  • Price-weighted: no rebalancing (weights = prices, self-updating)
  • Market-cap-weighted: minimal rebalancing (self-rebalances via price changes; only manual adjustments for corporate actions)
  • Equal-weighted: requires most frequent rebalancing
Reconstitution

Definition: Changing the constituent securities in the index (adding/removing securities).

  • Occurs on reconstitution dates — index provider reviews all constituents, re-applies inclusion criteria
  • Securities that no longer qualify are removed; new qualifying securities are added
  • Reflects market changes: bankruptcies, de-listings, mergers, acquisitions
  • Creates turnover in the index; requires weight adjustments for all remaining constituents
  • Style indexes have higher turnover than broad market indexes — stocks frequently migrate between style categories

Reconstitution — Market Impact (Russell 2000 Example)

The Russell 2000 reconstitutes annually. Because many funds use it as a benchmark, managers front-run reconstitution:

  • Buy stocks expected to be added → drives prices up before reconstitution date
  • Sell stocks expected to be removed → drives prices down before reconstitution date
  • Example: Uranium Energy Corporation (UEC) was anticipated to join Russell 2000 in 2009 → stock rose +300% before announcement, reaching +400% by reconstitution date
⚠️ Rebalancing vs. Reconstitution — Don't Confuse Them

Rebalancing = changing weights (same securities, different proportions). Reconstitution = changing the securities themselves (add/remove constituents). Both create turnover. Reconstitution happens less frequently but has much larger price impacts, especially for widely-followed indexes. Remember: reconstitution is part of the rebalancing cycle.

Uses of Security Market Indexes

💡 Mnemonic — 5 Uses of Indexes: "G-PABS"

Gauges of market sentiment · Proxies for measuring/modeling returns & risk · Asset class proxies for asset allocation models · Benchmarks for active portfolio managers · Seed model portfolios for index funds/ETFs

Use Description Key Detail / Example
Gauge of Market Sentiment Original purpose — indicator of collective market opinion/confidence DJIA: frequently quoted, long history, but only 30 stocks → not fully representative of all US equities
Proxy for Market Returns & Systematic Risk Used to represent market portfolio in CAPM; beta calculation; Sharpe/information ratio benchmarking TOPIX = proxy for Japanese market; S&P 500 = proxy for US market. Alpha = return of active portfolio − return of passive portfolio with same beta. Beta of active portfolio → construct equivalent passive benchmark (e.g., 95% MSCI World + 5% cash)
Asset Class Proxy (Asset Allocation Models) Indexes represent risk/return profiles of asset classes; provide historical data for modeling Enables mean-variance optimization — "what is the expected return and standard deviation of equities?" → use index historical data
Benchmark for Active Managers Compare active manager's returns to the index the manager's strategy most closely resembles Critical: benchmark must match strategy. Small-cap manager → FTSE Global Small Cap Index (not S&P 500). Wrong benchmark → wrong conclusion about manager skill. Manager might underperform S&P 500 but outperform small-cap benchmark → correct evaluation depends on proper benchmark choice
Model Portfolio for Index Products Indexes become the portfolio for index mutual funds and ETFs CAPM → market portfolio → broad index funds. ETFs then created for existing indexes; later new narrow indexes created specifically for new ETFs. E.g., VanEck Vietnam ETF → enables exposure to Vietnamese equities

Equity Indexes

Broad Market Indexes

Represent an entire equity market — typically >90% of selected market. Usually market-cap weighted.

  • SSE Composite: all Shanghai Stock Exchange stocks
  • Wilshire 5000: all US equities with available prices (entire US market)
  • Russell 3000: largest 3,000 US stocks ≈ 98% of US equity market
Multi-Market Indexes

Comprise indexes from multiple countries/regions — represent multiple security markets simultaneously. Can be global, regional, or developmental.

  • MSCI classifies: Developed, Emerging, Frontier markets
  • Geographic regions: Americas / Europe & Middle East / Asia-Pacific
  • MSCI ACWI covers 23 developed + 24 emerging markets
  • GDP-weighted multi-market indexes: weight countries by GDP (not market cap) → addresses concentration risk (e.g., Japan was 60% of MSCI EAFE in 1987)
Sector Indexes

Track specific economic sectors: consumer goods, energy, finance, health care, technology — on national, regional, or global basis.

  • Organized as families (sum of sector indexes = broad market index)
  • Used for performance attribution: was manager better at stock selection or sector allocation?
  • Serve as model portfolios for sector ETFs
  • No universally agreed classification method

Style Indexes

Represent securities classified by market cap, value/growth, or a combination. Reflect specific investing styles.

Market-Cap Categories
  • Large cap
  • Mid cap
  • Small cap

No universal definition — cutoffs vary by provider. Classification can be absolute (e.g., <€100M) or relative (e.g., smallest 2,500 stocks).

Value / Growth Classification

Providers use ratios to classify: low P/B, low P/E, high dividend yield → Value. High growth expectations → Growth. Different providers use different factors → same "small-cap value" label may have different constituents across providers.

6 Style Index Categories (3 Cap × 2 Style)
Large-Cap Value | Large-Cap Growth
Mid-Cap Value | Mid-Cap Growth
Small-Cap Value | Small-Cap Growth
Style indexes have HIGHER turnover than broad market indexes — stocks frequently migrate between categories as valuations and market caps change on reconstitution dates.

Fixed-Income Indexes

Construction Challenges — Why Fixed Income is Harder than Equity

  1. Enormous number of securities: fixed-income universe is many times larger than equities — governments, agencies, and corporations each issue many different securities with varying characteristics
  2. Pricing data availability: fixed-income markets are predominantly dealer markets → many securities don't trade frequently → index providers must contact dealers for prices OR estimate prices using similar traded securities
  3. Illiquidity: many bonds are illiquid → difficult and costly to replicate → index tracking error tends to be higher than for equity indexes
  4. Turnover from maturity: bonds mature regularly, creating constant turnover as they're replaced by new issues

Classification Dimensions for Fixed-Income Indexes

By Issuer / Type
  • Government (sovereign)
  • Government agency
  • Corporate
  • Collateralized / Securitized
  • Municipal
By Other Characteristics
  • Type of financing: general obligation vs. collateralized
  • Currency of payments
  • Maturity: short (1–3yr), intermediate (3–10yr), long (>10yr)
  • Credit quality: investment grade (BBB– or better) vs. high yield
  • Inflation protection: nominal vs. inflation-linked (TIPS)

Types of Fixed-Income Indexes

Type Description Key Example
Aggregate / Broad Market Represent an entire bond market across multiple sectors and credit ratings Bloomberg Barclays US Aggregate: ~8,000 securities — US Treasuries, agency, corporate, MBS, ABS, CMBS. First fixed-income index created
Market Sector Indexes Subdivisions of aggregate indexes by sector: government, agency, collateralized, corporate US Treasury Index, US Corporate Bond Index
Style Indexes Classified by maturity and/or credit quality Short-term (1–3yr) investment-grade; Long-term BBB index
High-Yield Indexes Sub-investment-grade bonds (below BBB– / Baa3) Markit iBoxx Euro High-Yield Bond Index (rebalanced monthly)
Inflation-Linked Indexes Inflation-protected securities (TIPS in US) Bloomberg Barclays US TIPS Index
Emerging Market Indexes Bonds issued by emerging market governments/corporations JP Morgan EMBI (Emerging Market Bond Index)
⚠️ Fixed-Income Replication is Harder than Equity

Because fixed-income indexes contain thousands of bonds, many of which are illiquid, exact replication is impractical. Fund managers typically use stratified sampling — hold a representative subset across maturity, credit quality, and sector cells — rather than buying every bond in the index. This is a key difference from equity index funds.

Indexes for Alternative Investments

Commodity Indexes

  • Consist of futures contracts on commodities (NOT spot/physical commodities directly)
  • Cover: agricultural products, livestock, precious metals, common metals, energy
  • No obvious weighting mechanism → providers create own methods:
    • TR/CC CRB Index: fixed number of commodities, equally weighted
    • S&P GSCI: combines liquidity measures + world production values; allocates more weight to higher-priced commodities; energy-heavy (~50% more than CRB)
    • Some use fixed weights determined by committee
  • Key issue: Commodity index returns ≠ spot price returns because futures must be "rolled over" (replace expiring contracts with new ones). Index return = risk-free rate + change in futures prices + roll yield. Roll yield can be negative (contango) or positive (backwardation)
  • Different weighting methods → large differences in commodity exposure → very different risk/return profiles. Unlike equity/bond indexes of same market (which converge) → two commodity indexes on same market can have very different returns

Real Estate Investment Trust (REIT) Indexes

  • Three types of real estate indexes:
    1. Appraisal indexes: based on periodic property valuations (infrequent)
    2. Repeat sales indexes: based on price changes of properties sold multiple times (e.g., Case-Shiller)
    3. REIT indexes: based on publicly traded REIT shares → continuous pricing
  • REIT indexes use shares of publicly traded REITs → calculated continuously (unlike appraisal/repeat sales which are infrequent)
  • REITs = corporations organized to invest in real estate (properties or mortgages); shares trade on stock exchanges
  • Key example: FTSE EPRA/NAREIT global REIT index family
  • REIT shares may also be constituents of broader equity market indexes

Hedge Fund Indexes

  • Reflect returns on hedge funds; maintained by research organizations from voluntary databases
  • Most are equal-weighted; represent performance within a database at global or strategy level
  • Critical problems with hedge fund indexes:
    1. Voluntary reporting: hedge funds are unregulated → not required to report to databases → each fund chooses which database(s) to report to → little overlap between indexes → different global hedge fund indexes can show very different performance for same period
    2. Survivorship bias: funds with poor performance are less likely to report or stop reporting → index overstates actual hedge fund performance (only surviving/successful funds tend to be included)
    3. Self-selection bias: constituents determine the index, not the index provider → index composition is not controlled
⚠️ Hedge Fund Index Survivorship Bias — High-Frequency Exam Topic

Survivorship bias causes hedge fund indexes to overstate true industry performance. Funds that blow up quietly stop reporting. Only the survivors (or the strong performers who choose to publicize results) appear in the index. This makes hedge fund benchmarking fundamentally different from equity/bond benchmarking and a go-to exam topic. Also: because constituents determine the index (not the provider), different databases have little overlap → two "global" hedge fund indexes may reflect vastly different performance.

Last-Minute Exam Checklist

  • ✅ Price return index = prices only. Total return index = prices + reinvested income. At inception they are equal; total return index grows faster over time.
  • ✅ Divisor: set at inception for convenient starting value; adjusted for non-price events (stock splits, constituent changes) to preserve index continuity. New divisor after split = Σ post-split prices / index value before split.
  • ✅ Price return of index = (V_PRI1 − V_PRI0) / V_PRI0 = Σ wᵢ × PRᵢ (weighted average of constituent price returns).
  • ✅ Total return of index = (V_PRI1 − V_PRI0 + Inc_I) / V_PRI0 = Σ wᵢ × TRᵢ. Denominator is V_PRI0 (beginning price index value).
  • ✅ Total return of security: TRᵢ = (P1ᵢ − P0ᵢ + Incᵢ) / P0ᵢ. Don't forget to add income to numerator.
  • ✅ Multi-period calculation: geometric linking only — V_T = V_0 × (1+R₁)(1+R₂)...(1+Rₙ). NEVER simple arithmetic addition of returns.
  • ✅ 5 index construction decisions: target market → security selection → weighting method → rebalancing schedule → reconstitution schedule.
  • ✅ Price weighting: wᵢ = Pᵢ / Σ P. High-price stocks dominate. Stock split → adjust divisor. No rebalancing needed. Arbitrary weights. Examples: DJIA, Nikkei 225.
  • ✅ Equal weighting: wᵢ = 1/N. Simple; small-cap tilted; large caps underrepresented; requires most frequent rebalancing (highest turnover). Examples: CRB Index, HFRX Equal Weighted.
  • ✅ Market-cap weighting: wᵢ = QᵢPᵢ / Σ QP. Reflects actual market weight; largely self-rebalancing. Disadvantage: momentum effect (overweights rising/potentially overvalued stocks). Most major indexes use this (S&P 500, MSCI World, FTSE 100).
  • ✅ Float-adjusted market-cap weighting: uses fᵢQᵢPᵢ in numerator — excludes closely held shares (controlling shareholders, corporations, governments, foreign-restricted shares). Most modern indexes are float-adjusted.
  • ✅ Fundamental weighting: wᵢ = Fᵢ / Σ F (F = book value, earnings, revenues, dividends, etc.). Value tilt + contrarian effect (trims winners, adds to losers at rebalancing). Opposite of market-cap momentum effect.
  • ✅ Momentum effect = market-cap weighted. Contrarian effect = fundamentally weighted. This is a very common exam question.
  • ✅ Rebalancing = adjusting weights of existing constituents back to targets (usually quarterly). Reconstitution = changing which securities are in the index (less frequent; bigger market impact). Both create turnover.
  • ✅ Style indexes have higher turnover than broad market indexes — stocks migrate across cap and value/growth categories frequently on reconstitution dates.
  • ✅ 5 uses of indexes (G-PABS): Gauge market sentiment, Proxy for market returns/risk, Asset class proxy for allocation models, Benchmark for active managers, Seed for index funds/ETFs.
  • ✅ Benchmark must match strategy: small-cap manager → small-cap benchmark. Wrong benchmark → wrong performance conclusion. Alpha = active return − passive return with same systematic risk.
  • ✅ Equity index types: Broad market (>90% of market), Multi-market (multiple countries; MSCI ACWI = 23 developed + 24 emerging), Sector (economic sectors; families sum to broad market), Style (6 categories: 3 cap × 2 style).
  • ✅ MSCI market classifications: Developed → Emerging → Frontier. MSCI periodically reviews and reclassifies countries.
  • ✅ GDP-weighting in multi-market indexes: weights countries by GDP rather than market cap → reduces concentration in large markets (e.g., cut Japan's 60% EAFE weight in half in 1987).
  • ✅ Fixed-income index challenges: (1) huge number of securities, (2) dealer market → limited price transparency, (3) illiquidity → need price estimation, (4) maturity-driven turnover. Harder to replicate than equity indexes.
  • ✅ Bloomberg Barclays US Aggregate: first fixed-income index; ~8,000 securities; Treasuries + agency + corporate + MBS + ABS + CMBS. Investment-grade only.
  • ✅ Fixed-income classifications: by issuer type, financing type, currency, maturity, credit quality (investment grade = BBB– or better), inflation protection.
  • ✅ Commodity indexes: based on futures contracts (NOT spot prices). Returns = risk-free rate + futures price change + roll yield. Roll yield can be positive or negative. Different commodity indexes on same market can have very different returns due to different weighting methods (unlike equity/bond indexes).
  • ✅ REIT index types: Appraisal (infrequent) → Repeat sales (periodic) → REIT index (continuous — publicly traded shares). REIT indexes calculated continuously unlike other real estate indexes.
  • ✅ Hedge fund index problems: (1) Voluntary reporting → little overlap between databases → different indexes show different performance for same period. (2) Survivorship bias → overstates performance (poor performers stop reporting). (3) Constituents determine index, not index provider.
  • ✅ DJIA: 30 stocks, price-weighted, Wall Street Journal editor-selected, oldest US equity index. S&P 500: 500 stocks, float-adjusted market cap, committee-selected. Russell 2000/3000: market cap, annually reconstituted → front-running effect. TOPIX: all Tokyo SE First Section stocks, float-adjusted market cap.

V_PRI = Σ(nᵢ × Pᵢ) / D

PR_I = (V_PRI1 − V_PRI0) / V_PRI0 = Σ wᵢ × PRᵢ
PRᵢ = (Pᵢ1 − Pᵢ0) / Pᵢ0

TR_I = (V_PRI1 − V_PRI0 + Inc_I) / V_PRI0 = Σ wᵢ × TRᵢ
TRᵢ = (P1ᵢ − P0ᵢ + Incᵢ) / P0ᵢ

Multi-period: V_T = V_0 × (1+R₁)(1+R₂)...(1+Rₙ) [geometric linking]

Price weight: wᵢᴾ = Pᵢ / Σ Pᵢ
Equal weight: wᵢᴱ = 1 / N
Market-cap weight: wᵢᴹ = (Qᵢ × Pᵢ) / Σ (Qⱼ × Pⱼ)
Float-adj mkt cap: wᵢᴹ = (fᵢ × Qᵢ × Pᵢ) / Σ (fⱼ × Qⱼ × Pⱼ)
Fundamental weight: wᵢᶠ = Fᵢ / Σ Fⱼ
For divisor adjustment after stock split: New D = Σ prices after split / index value before split
Weights always sum to 1.0 (100%). All returns expressed as decimals in formulas.
