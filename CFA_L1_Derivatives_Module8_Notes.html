Options vs. Forward Commitments — The Core Distinction

Options are contingent claims — they give one party (the buyer) the right but not the obligation to transact. This is fundamentally different from a forward commitment where both parties are obligated. Understanding this asymmetry is the foundation of everything in this module.

The option buyer pays an upfront premium (c₀ for a call, p₀ for a put) to the seller in exchange for this right. The seller receives the premium and takes on the obligation to perform if exercised.

📋 Forward Commitment
  • Firm commitment — both sides obligated
  • No premium at inception — V₀ = 0
  • Symmetric payoff: linear, one-for-one with underlying
  • Settle at maturity regardless of direction
  • Priced by no-arbitrage using forward price F₀(T)
  • Value at maturity: V_T = S_T − F₀(T) for long
⚡ Contingent Claim (Option)
  • Buyer has the right; seller has the obligation
  • Premium paid upfront (c₀ or p₀) — cost to buyer
  • Asymmetric payoff: non-linear — only exercises when profitable
  • Settles only if in the money at maturity
  • Priced with no-arbitrage bounds (upper and lower)
  • Value at maturity: Max(0, S_T − X) for call; Max(0, X − S_T) for put
Option Payoff & Profit at Maturity (T)
CALL OPTION BUYER:
Payoff = Max(0, S_T − X)
Profit = Max(0, S_T − X) − c₀ [c₀ = premium paid at t=0]

PUT OPTION BUYER:
Payoff = Max(0, X − S_T)
Profit = Max(0, X − S_T) − p₀ [p₀ = premium paid at t=0]

KEY: Option buyer's maximum loss = premium paid (c₀ or p₀)
Option seller's maximum gain = premium received
Option seller's maximum loss = unlimited (call seller) or X − c₀ net (put seller)
Why Max(0, ...)? The option buyer will only exercise if payoff is positive. If S_T < X for a call, the buyer simply walks away — the option expires worthless. The Max function captures this asymmetry.

Profit vs. Payoff: Payoff ignores the premium. Profit accounts for the premium cost. The break-even for a call buyer is S_T = X + c₀. For a put buyer: S_T = X − p₀.
💡 European vs. American Options — This Module's Scope

European options: Can only be exercised AT maturity (t = T). This module focuses entirely on European options.
American options: Can be exercised at ANY time before or at maturity. Same directional sensitivities, but American options are always worth ≥ their European equivalents (early exercise right has value). The exam may ask you to recognize this relationship.

02

Exercise Value (Intrinsic Value) — Equations 1 & 2

Exercise Value (= Intrinsic Value)
The value of an option if it were immediately exercisable at time t. It equals the current payoff of the option, incorporating the time value of money on the exercise price. This is NOT the full option price — it excludes time value. The exercise value is always ≥ 0 (an option can't have negative exercise value — the buyer just wouldn't exercise).
Equation 1 — Call Option Exercise Value at time t
Call Exercise Value = Max(0, S_t − X(1+r)^−(T−t))
= Max(0, S_t − PV(X))

Where:
S_t = spot price of underlying at time t
X = exercise (strike) price
r = risk-free rate
T−t = remaining time to maturity
PV(X) = X(1+r)^−(T−t) = present value of the exercise price
Why PV(X) and not just X? The option doesn't require payment of X until maturity (T). So the "cost" of buying the underlying via the option in today's terms is only the PV of X — not the full X. This makes the exercise value larger than the simple (S_t − X) difference.

Link to forward value: If X = F₀(T), the call exercise value equals the value of a long forward at time t: V_t(T) = S_t − PV(F₀(T)) = S_t − PV(X). The call and forward have the same exercise value when X = F₀(T) — but the option buyer paid a premium; the forward required nothing upfront.
Equation 2 — Put Option Exercise Value at time t
Put Exercise Value = Max(0, X(1+r)^−(T−t) − S_t)
= Max(0, PV(X) − S_t)

Intuition: The put is the right to SELL the underlying at X.
Benefit = receiving PV(X) at maturity → worth PV(X) today
Cost = giving up the underlying worth S_t today
Net = PV(X) − S_t (positive = in the money for put)
Key insight: The put exercise value mirrors the call, but with X and S_t swapped. Calls benefit when S_t > PV(X). Puts benefit when PV(X) > S_t (i.e., when S_t is low).

Worked Example — Put Option Exercise Value (Example 1)

1-year put | X = EUR 1,000 | r = 1% | t = 6 months (T−t = 0.5 years) | S_t = EUR 950
PV(X) = 1,000 × (1.01)^−0.5 = 1,000 × 0.99504 = EUR 995.04

Put Exercise Value = Max(0, PV(X) − S_t)
= Max(0, 995.04 − 950)
= Max(0, 45.04)
= EUR 45.04 ← the put is IN the money

Note: Simple (X − S_t) = 1,000 − 950 = 50. Exercise value (45.04) is LESS than 50
because PV(X) < X — we won't receive X until maturity, not today.
⚠️ Common Exam Trap — Using X Instead of PV(X)

Students often calculate exercise value as (S_t − X) for a call or (X − S_t) for a put. This is wrong before maturity. The correct formula uses PV(X) = X(1+r)^−(T−t) because the exercise price isn't paid/received until maturity. At maturity (t = T), (T−t) = 0, so PV(X) = X and the formulas reduce to the simple Max(0, S_T − X) and Max(0, X − S_T). The only time you use X directly (not PV(X)) is at maturity itself.

03

Option Moneyness — ITM, ATM, OTM

Moneyness describes the relationship between the underlying spot price (S_t) and the exercise price (X). It tells you whether and by how much an option would pay off if exercised right now. It's used to compare options on the same underlying with different strike prices and/or maturities.

Moneyness StatusCall Option (c_t)Put Option (p_t)Intuition
In the Money (ITM)S_t > XS_t < XOption has positive exercise value; likely to be exercised if at maturity
At the Money (ATM)S_t = XS_t = XExercise value = 0; small underlying moves determine whether it'll be exercised
Out of the Money (OTM)S_t < XS_t > XExercise value = 0; unlikely to be exercised; still has time value
💡 Memory Trick — Which Way Is "In the Money"?

Call = you want to BUY cheap. A call is ITM when S_t > X — you can buy at the low exercise price X when the market is above it. You're "in the money" because you get a below-market deal.

Put = you want to SELL high. A put is ITM when S_t < X — you can sell at the high exercise price X when the market is below it. You're "in the money" because you get an above-market sale.

Mnemonic: Call = CLIMB (wants S_t to go up). Put = PLUNGE (wants S_t to go down).

Moneyness and Price Sensitivity

The degree of moneyness affects how sensitively the option's price responds to changes in the underlying price:

  • Deep ITM: Near 1:1 correspondence — option price moves almost dollar-for-dollar with the underlying. The option behaves like the underlying itself.
  • ATM: Most sensitive threshold — even a tiny move in S_t can flip the option from OTM to ITM (or vice versa). Large time value here.
  • Deep OTM: Very little sensitivity — a given change in the underlying barely moves the option's price. Mostly time value, very little exercise value.

This non-linear sensitivity is fundamentally different from forward contracts (which have constant 1:1 sensitivity). In later modules this sensitivity is formalized as delta (Δ).

Example 2 — Put Option Moneyness Check
From Example 1: t = 6 months remaining, X = EUR 1,000, S_t = EUR 950

For a PUT: ITM when S_t < X
S_t = 950 < X = 1,000 → PUT IS IN THE MONEY
Amount ITM = X − S_t = 1,000 − 950 = EUR 50 (simple moneyness measure)
Exercise value = EUR 45.04 (incorporates time value of money on X)

Note: Moneyness is typically stated as (X − S_t) = EUR 50 "in the money"
(the simple difference, not using PV(X)). Exercise value uses PV(X).
04

Option Time Value & Time Value Decay — Equations 3 & 4

Time Value of an Option
The portion of an option's price that exceeds its exercise value. It represents the additional value from the possibility that the underlying price will move favorably before maturity. Time value is always ≥ 0 (an option's price is always ≥ its exercise value). At maturity, time value = 0.
Equation 3 — Call Option: Price = Exercise Value + Time Value
c_t = Max(0, S_t − X(1+r)^−(T−t)) + Time Value

Rearranging:
Call Time Value = c_t − Max(0, S_t − PV(X))

Since c_t ≥ Max(0, S_t − PV(X)) always:
Time Value ≥ 0 always
Equation 4 — Put Option: Price = Exercise Value + Time Value
p_t = Max(0, X(1+r)^−(T−t) − S_t) + Time Value

Rearranging:
Put Time Value = p_t − Max(0, PV(X) − S_t)

Summary: Option Price = Exercise Value + Time Value (for both calls and puts)
Example 3 — Put Time Value Calculation
From previous: Put exercise value = EUR 45.04 (with 6 months remaining)
Observed put option price: p_t = EUR 50

Put Time Value = p_t − Exercise Value
= 50 − 45.04
= EUR 4.96

Interpretation:
• EUR 45.04 → you'd get right now if you could exercise (exercise value)
• EUR 4.96 → extra amount you pay for the chance rates move further in your favor
• EUR 50.00 → total premium (market price of the put)

Time Value Decay — The Key Behavior

Time value decreases as the option approaches maturity — a process called time value decay (also called "theta decay"). This happens because:

  • Less time remaining = less chance for favorable underlying price moves
  • At maturity: time value = 0. Option price = exercise value only = Max(0, S_T − X) or Max(0, X − S_T)
  • Time value decay is not linear — it accelerates as expiry approaches (especially for ATM options)

The decay path: Deep ITM options → time value is small (option already has high exercise value; little additional "optionality"). Deep OTM options → time value is small (unlikely to be exercised). ATM options → time value is highest (most "optionality"; S_t is right at the inflection point).

💡 What Creates Time Value?
  • Time to maturity: More time = more chance for favorable moves = higher time value
  • Volatility: More volatility = wider range of outcomes = higher time value
  • Both factors work together — and they always increase both call AND put time values
⚠️ Time Value vs. Time Value of Money
  • These are completely different concepts
  • Option time value = the speculative/optionality component of option price
  • Time value of money = r, the risk-free rate — used in discounting PV(X)
  • The risk-free rate (r) does NOT directly affect option time value (per the LES)

No-Arbitrage Price Bounds — Equations 9 through 14

Because options have asymmetric payoffs, we cannot pin down an exact price using simple no-arbitrage (as we do for forwards). Instead, we establish upper and lower bounds within which the option price must lie. Any option trading outside these bounds creates a riskless arbitrage profit opportunity.

The key insight: an option that trades below its exercise value violates no-arbitrage — you could buy the option and immediately exercise for a riskless profit.

Call Option Bounds — Equations 9, 10, 11

Call Option No-Arbitrage Bounds
LOWER BOUND (Eq. 10): c_t,Lower = Max(0, S_t − X(1+r)^−(T−t))
= Max(0, S_t − PV(X))
= Exercise value of the call

UPPER BOUND (Eq. 11): c_t,Upper = S_t
(Call buyer will NEVER pay more than the underlying itself)

COMBINED (Eq. 9): Max(0, S_t − PV(X)) < c_t ≤ S_t
Why is the lower bound the exercise value? If c_t < Max(0, S_t − PV(X)), you can buy the call and immediately exercise (or synthetically exercise) to lock in a riskless profit. No rational seller would sell below exercise value.

Why is the upper bound S_t? A call gives you the RIGHT to buy the underlying. Why would you pay MORE than the underlying price itself for that right? You'd just buy the underlying directly for S_t. So c_t ≤ S_t always.

Put Option Bounds — Equations 12, 13, 14

Put Option No-Arbitrage Bounds
LOWER BOUND (Eq. 13): p_t,Lower = Max(0, X(1+r)^−(T−t) − S_t)
= Max(0, PV(X) − S_t)
= Exercise value of the put

UPPER BOUND (Eq. 14): p_t,Upper = X
(Best case for put: underlying → 0; receive X at exercise; PV is at most X)

COMBINED (Eq. 12): Max(0, PV(X) − S_t) < p_t ≤ X
Why is the upper bound X (not PV(X))? The convention is to use X as the upper bound for puts. The absolute maximum payoff on a put is X (when the underlying goes to zero). So the put can never be worth more than X.

Why not PV(X)? The upper bound uses X, not PV(X), because we're bounding the current option price conservatively — X is always ≥ PV(X), making it a valid (if slightly loose) upper bound.

Compare to call upper bound: Call upper = S_t (underlying price, no discounting). Put upper = X (exercise price, no discounting). These are the maximums each could ever be worth.

Worked Example — Call Option Bounds (Example 4)

1-year call | X = EUR 1,000 | r = 1% | at t = 6 months: S_t = EUR 1,050
LOWER BOUND:
PV(X) = 1,000 × (1.01)^−0.5 = EUR 995.04
c_t,Lower = Max(0, S_t − PV(X)) = Max(0, 1,050 − 995.04) = Max(0, 54.96) = EUR 54.96

UPPER BOUND:
c_t,Upper = S_t = EUR 1,050

CONCLUSION: EUR 54.96 < c_t ≤ EUR 1,050
The call must be priced between EUR 54.96 and EUR 1,050.
Any price below EUR 54.96 → arbitrage profit available.
Any price above EUR 1,050 → you'd just buy the underlying instead.
BoundCall (c_t)Put (p_t)Reason
Lower BoundMax(0, S_t − PV(X))Max(0, PV(X) − S_t)= exercise value; trading below = arbitrage
Upper BoundS_t (underlying price)X (exercise price)Can't rationally pay more than this
At inception (t=0)Max(0, S₀ − PV(X))Max(0, PV(X) − S₀)Same formulas; T−t = T at inception
At maturity (t=T)Max(0, S_T − X)Max(0, X − S_T)PV(X) = X when T−t = 0
⚠️ The Inequality is Strict on the Lower Bound

The equations use strict inequality for the lower bound: c_t > Max(0, S_t − PV(X)), not ≥. This is because if c_t exactly equals the lower bound, there is no time value — implying the option will definitely be exercised or definitely won't be. For any option with time remaining, there must be positive time value, so the actual price exceeds the exercise value. At maturity only does the price exactly equal the exercise value (time value = 0).

Replication of Options — Key Differences from Forward Replication

Just as forward commitments can be replicated using a combination of the underlying asset and borrowing/lending, options can also be replicated. However, the asymmetric payoff profile of options introduces a crucial difference: the replication must be dynamically adjusted over time as the likelihood of exercise changes.

Call Option Replication

Long Forward Replication (Fixed)
  • At t=0: Borrow PV(F₀(T)) = PV(X), buy underlying at S₀
  • At T: Sell underlying for S_T, repay X
  • Net payoff: S_T − X (always — both gain and loss)
  • Replicating trade is FIXED — no adjustment needed
Long Call Option Replication (Dynamic)
  • At t=0: Borrow a proportion of PV(X); buy underlying
  • At T (if ITM): Sell underlying for S_T, repay X → payoff = S_T − X
  • At T (if OTM): No exercise → no settlement
  • Replicating trade must be ADJUSTED as exercise likelihood changes
Call Replication — The Core Logic
FORWARD replication at t=0: Borrow exactly PV(X), buy underlying S₀
→ At T: Always exercise → payoff = S_T − X

CALL replication at t=0: Borrow (proportion × PV(X)), buy (proportion × underlying)
→ The "proportion" is based on the LIKELIHOOD of exercise
→ This proportion changes as the option moves ITM/OTM/ATM
→ Deep ITM call: proportion ≈ 1 (behaves like a forward)
→ Deep OTM call: proportion ≈ 0 (barely any underlying held)
→ ATM call: proportion ≈ 0.5

This proportion is formally known as DELTA (Δ) in later lessons.
Key exam point: For forward replication — static (no adjustment). For option replication — dynamic (must be continuously adjusted). This is why option pricing is more complex and requires models like Black-Scholes-Merton.

Put Option Replication

To replicate a long put option at inception (t=0):

  • Sell the underlying SHORT at price S₀
  • Lend the proceeds at the risk-free rate r
  • This gives you a position that benefits when S_T falls (just like a put)

At expiration, two possible outcomes:

  • Exercise (S_T < X): Buy the underlying for S_T from the risk-free loan proceeds → payoff = X − S_T
  • No exercise (S_T > X): No settlement. Close short position at a loss offset by the loan proceeds.

As with the call, the amount lent is a proportion of PV(X) based on the likelihood of exercise — not the full PV(X) as would be the case for a forward. And as the likelihood of exercise changes, the replicating position must be dynamically adjusted.

FeatureForward Commitment ReplicationOption Replication
Underlying positionFull long (call) or full short (put)Proportional long/short (based on exercise likelihood)
Borrowing / lendingFull PV(X) at inceptionProportion of PV(X) at inception
Adjustment over timeNone — static replicationContinuous — dynamic replication required
What determines proportionN/A (always 1:1)Moneyness / likelihood of exercise (= delta)
Payoff at maturityS_T − F₀(T) for long (symmetric)Max(0, S_T − X) for call; Max(0, X − S_T) for put
💡 Why Does the Replication Proportion Change?

As the underlying price moves, the probability of exercise changes. A deep OTM option is very unlikely to be exercised → replication holds almost no underlying. A deep ITM option is almost certain to be exercised → replication holds nearly the full underlying. An ATM option is 50/50 → about half the underlying. As the underlying price moves continuously, the replicating portfolio must be continuously readjusted. This is delta hedging — a concept developed further in later modules.

Six Factors That Determine Option Value

Six factors determine the value of an option. Two factors have the same directional effect on both calls and puts (time to maturity, volatility). Four factors have opposite effects on calls vs. puts (underlying value, exercise price, risk-free rate, income/costs). Memorizing which is which is a high-frequency exam topic.

① Value of the Underlying (S_t)

📈 Call Option — S_t rises → Value INCREASES

A higher underlying price means S_t is further above X → higher exercise value for ITM calls, or the call is pushed into the money. More favorable → call value rises. Same direction as a long forward position.

📉 Put Option — S_t rises → Value DECREASES

A higher underlying price means S_t moves away from (and above) X → lower exercise value for puts, or the put is pushed out of the money. Less favorable → put value falls. Same direction as a short forward position.

⚠️ Non-Linear Relationship — Options vs. Forwards

For a forward: value changes 1:1 with the underlying (linear). For an option: the sensitivity depends on moneyness. Deep ITM → ~1:1. ATM → ~0.5:1. Deep OTM → ~0:1. This non-linearity (captured by delta) is a key distinction between contingent claims and forward commitments.

② Exercise Price (X)

Call — Lower X → Higher Value

Lower X means a lower threshold to buy the underlying → more likely to be exercised AND higher payoff when exercised. Max(0, S_T − X) rises when X falls. X is a lower bound on call exercise value.

Put — Higher X → Higher Value

Higher X means a higher price at which you can sell the underlying → more likely to be exercised AND higher payoff when exercised. Max(0, X − S_T) rises when X rises. X is an upper bound on put exercise value.

③ Time to Expiration (T − t)

Effect on BOTH calls and puts: generally positive (more time = more value) — but with an important exception for puts.

Call — More Time → ALWAYS Higher Value

A longer time horizon means more opportunity for S to rise above X. The upside is unlimited; the downside is capped at the premium. Longer time always increases call value — no exceptions for European calls.

Put — More Time → Usually Higher Value (Exception Exists)

More time allows S to fall further below X → generally good. BUT: longer time also reduces PV(X) (the payoff is discounted for longer). For a deep ITM put with high r, this discounting effect can dominate → put value may DECREASE with more time. This exception only occurs in specific conditions (deep ITM, high r).

💡 Exam Note — Put Time to Expiry is "+/−"

The factor table shows "+/−" for put time to expiration. The examiners expect you to know that while the answer is usually positive, it can be negative for deep-in-the-money puts when the risk-free rate is high and time to expiry is long. The reason: the put buyer waits to receive X − S_T at maturity. A longer wait means PV(X − S_T) is smaller. If the option is already deep ITM and highly likely to be exercised, the time value of waiting for exercise is a negative, not a positive.

④ Risk-Free Interest Rate (r)

Call — Higher r → Higher Value

Higher r → lower PV(X) → lower cost to "buy" the underlying via the option (you pay PV(X) now in equivalent terms). Call exercise value = S_t − PV(X). If PV(X) falls, exercise value rises. Higher r = better for call.

Put — Higher r → Lower Value

Higher r → lower PV(X) → lower value of being able to sell at X. Put exercise value = PV(X) − S_t. If PV(X) falls, exercise value falls. Higher r = worse for put.

Intuition — r and PV(X)
PV(X) = X × (1+r)^−(T−t)

As r ↑ → PV(X) ↓

Call exercise value = S_t − PV(X) → rises when PV(X) falls → CALL ↑
Put exercise value = PV(X) − S_t → falls when PV(X) falls → PUT ↓

Note: r does NOT affect TIME VALUE of the option directly —
its effect works through the PV of the exercise price only.

⑤ Volatility of the Underlying (σ)

Volatility ALWAYS increases BOTH call and put values — same direction for both. This is the most unique option factor (no direct analog in forward pricing).

★ Volatility — The Most Important Option-Specific Factor

Higher volatility = wider dispersion of future underlying prices = higher probability of large favorable moves. The option buyer captures all the upside (when S_T > X for a call, or S_T < X for a put) but ignores the downside (option simply expires worthless — loss capped at premium). So:

  • Higher σ → wider range of possible outcomes → more chance of a large favorable move → both call AND put values rise
  • Lower σ → narrower range → less chance of large moves → time value falls for both
  • Why doesn't downside volatility hurt options? Because the option buyer doesn't have to participate in unfavorable outcomes (they just let the option expire)
💡 Memory Trick — Volatility Always Helps Option BUYERS

Options give you asymmetric exposure — you capture the gains from volatility without being exposed to the losses (beyond the premium). Think of it this way: a call option on a very volatile stock has a much higher chance of ending up ITM (even starting OTM) than a call on a stable stock with the same X. The buyer always benefits from volatility. The seller of options hates volatility (that's why they demand a higher premium when σ is high).

⑥ Income or Cost Related to Owning the Underlying

Income/Benefits (Dividends, Convenience Yield)
  • Call value DECREASES: Income accrues to the owner of the underlying, NOT to the option holder. A stock paying dividends is effectively worth less from the call buyer's perspective (the price drop on ex-div date reduces S_t). Lower S_t → lower call value.
  • Put value INCREASES: Same reason — the put buyer benefits because income reduces S_t, pushing the put further ITM.
Carry Costs (Storage, Insurance for Commodities)
  • Call value INCREASES: Owning the underlying costs money (storage, insurance). The call buyer avoids these costs by using an option instead. This makes the call more attractive → higher call value.
  • Put value DECREASES: Same costs reduce the attractiveness of owning the underlying (which you must then sell at X on a put) → lower put value.
💡 Memory Trick — Income vs. Costs

Income (dividends): Drains value from the underlying → Call ↓ (loses the income), Put ↑ (underlying worth less → put more ITM).
Carry costs (storage): Adds a burden to owning → Call ↑ (avoids the cost), Put ↓ (underlying burden makes it less attractive to own and sell later).
Think: income and costs have opposite effects on calls vs. puts. And income vs. costs have opposite effects on each other.

Exam Checklist & Quick Reference Summary

  • ✅ Option = contingent claim. Buyer pays premium for the RIGHT (not obligation) to transact. Seller receives premium and has the obligation. Max loss for buyer = premium. Max gain for seller = premium.
  • ✅ Payoff at maturity: Call = Max(0, S_T − X). Put = Max(0, X − S_T). Profit = Payoff − Premium paid (ignoring time value of money on premium).
  • ✅ Call Exercise Value (Eq. 1): Max(0, S_t − PV(X)) = Max(0, S_t − X(1+r)^−(T−t)). Uses PV(X), NOT just X. At maturity only: PV(X) = X.
  • ✅ Put Exercise Value (Eq. 2): Max(0, PV(X) − S_t) = Max(0, X(1+r)^−(T−t) − S_t). Mirrors call but signs reversed.
  • ✅ Exercise value ≥ 0 always. An option can't have negative exercise value — buyer simply won't exercise. The Max(0,...) captures this floor.
  • ✅ Time value = Option price − Exercise value. Time value ≥ 0 always. At maturity, time value = 0 (price = exercise value exactly).
  • ✅ Time value decay: Time value falls as option approaches maturity. Non-linear — accelerates near expiry. Highest time value for ATM options.
  • ✅ Moneyness — Call: ITM if S_t > X; ATM if S_t = X; OTM if S_t < X. Put: ITM if S_t < X; ATM if S_t = X; OTM if S_t > X.
  • ✅ Deeper ITM → ~1:1 price sensitivity to underlying. Deeper OTM → ~0 sensitivity. ATM → ~0.5 sensitivity. This non-linearity contrasts with forward commitment's constant 1:1 relationship.
  • ✅ Call bounds (Eqs. 9–11): Max(0, S_t − PV(X)) < c_t ≤ S_t. Lower = exercise value. Upper = underlying price.
  • ✅ Put bounds (Eqs. 12–14): Max(0, PV(X) − S_t) < p_t ≤ X. Lower = exercise value. Upper = exercise price X (not PV(X)).
  • ✅ Strict lower bound inequality: c_t > exercise value (not ≥) before maturity because time value > 0. Equality only at maturity.
  • ✅ Replication — call: Buy proportion of underlying + borrow proportion of PV(X). Put: Short sell proportion of underlying + lend proportion of PV(X). Proportion = likelihood of exercise (= delta).
  • ✅ Replication is DYNAMIC for options (must be adjusted continuously as moneyness changes). Replication is STATIC for forwards (no adjustment needed). This is the key structural difference.
  • ✅ Factor: S_t ↑ → Call ↑, Put ↓. Non-linear relationship (unlike forwards).
  • ✅ Factor: X ↑ → Call ↓, Put ↑. Lower X = better for call (lower cost to buy). Higher X = better for put (higher price to sell).
  • ✅ Factor: Time to expiry ↑ → Call ↑ (always), Put ↑ (usually, exception: deep ITM put + high r).
  • ✅ Factor: r ↑ → Call ↑, Put ↓. Higher r → lower PV(X) → raises call exercise value, lowers put exercise value. Does NOT directly affect time value.
  • ✅ Factor: σ (volatility) ↑ → BOTH Call ↑ AND Put ↑. This is the only factor with the same positive direction for both. Higher σ = wider outcome range = more optionality value for buyer (downside capped at premium).
  • ✅ Factor: Income/benefits ↑ → Call ↓, Put ↑. Income accrues to underlying owner, not option holder. Reduces effective S_t. Bad for call, good for put.
  • ✅ Factor: Carry costs ↑ → Call ↑, Put ↓. Costs of owning underlying. Call avoids these costs (better). Put must own and then sell at X (worse, burdened by costs).
  • ✅ Same direction for both: σ and T. Opposite direction (call vs. put): S_t, X, r, income, costs.
  • ✅ European vs. American: American ≥ European value always (early exercise right). This module covers European options only.

═══ KEY FORMULAS ═══
Call exercise value: Max(0, S_t − X(1+r)^−(T−t)) [Eq. 1]
Put exercise value: Max(0, X(1+r)^−(T−t) − S_t) [Eq. 2]
Option price: Exercise value + Time value
Call time value: c_t − Max(0, S_t − PV(X)) [Eq. 3]
Put time value: p_t − Max(0, PV(X) − S_t) [Eq. 4]

═══ NO-ARBITRAGE BOUNDS ═══
Call: Max(0, S_t − PV(X)) < c_t ≤ S_t
Put: Max(0, PV(X) − S_t) < p_t ≤ X

═══ MONEYNESS ═══
ITM Call: S_t > X | ITM Put: S_t < X
ATM: S_t = X (both)
OTM Call: S_t < X | OTM Put: S_t > X

═══ FACTOR EFFECTS (increase in factor → effect on option value) ═══
S_t ↑: Call ↑ | Put ↓
X ↑: Call ↓ | Put ↑
T−t ↑: Call ↑ | Put +/− (usually ↑)
r ↑: Call ↑ | Put ↓
σ ↑: Call ↑ | Put ↑ ← SAME direction
Income ↑: Call ↓ | Put ↑
Carry cost↑: Call ↑ | Put ↓

═══ REPLICATION ═══
Call: Long (proportion) of underlying + Borrow (proportion × PV(X))
Put: Short (proportion) of underlying + Lend (proportion × PV(X))
Proportion = likelihood of exercise = delta (Δ) — changes dynamically

═══ OPTIONS vs. FORWARDS ═══
Forward: no premium, symmetric, static replication, exact price
Option: premium paid, asymmetric, dynamic replication, price bounds
Both: no-arbitrage framework, underlying drives value
Volatility is the most option-specific factor — it has no analog in forward pricing and always helps both buyers simultaneously. This is because options provide asymmetric participation in the distribution of outcomes.
Risk-free rate effect works through PV(X) only — not through the time value component. Higher r → lower PV(X) → call exercise value ↑, put exercise value ↓.
At maturity: time value = 0. Option price collapses to its payoff: Max(0, S_T − X) for call; Max(0, X − S_T) for put. PV(X) = X at T.
